## Round 1: 初始审稿

### 评分：4/10（Weak Reject）

### Strengths
1. 问题重要且及时，跨域FL是真实难题
2. 挑战了"域/风格信息应被移除"的常见假设
3. "风格作为可共享资产"的framing有趣
4. 按需调度比无差别共享更有说服力
5. 实验计划覆盖了有意义的benchmark

### Weaknesses
1. **新颖性过度声称** — 方法是已知成分的组合，不是新原理
2. **解耦声明弱** — 正交+HSIC不能严格保证"语义"和"风格"的识别，可能只是任意特征分裂
3. **风格原型定义模糊** — 是per-class还是per-domain？类条件的风格不是纯风格
4. **语义原型仍可能模糊** — 全局聚合的语义原型在强域偏移下同样会退化
5. **隐私问题严重缺失** — 风格共享可能泄露客户端/域身份
6. **模块过多** — 双头、正交、HSIC、语义对比、风格对比、BN、风格仓库、去重、调度、MixStyle → 收益可能来自额外容量而非核心思想
7. **可扩展性不明** — HSIC在minibatch中不稳定，仓库机制多类/多域下开销大
8. **标签保持未保证** — 外部风格注入可能破坏稀有类

### 8个必须回答的问题
1. 风格原型的精确定义是什么？（per-class / per-domain / global cluster）
2. 如何验证z_sty是风格主导而不携带类语义？
3. 为什么正交+HSIC足够做语义/风格解纠缠而不是任意分裂？
4. 风格共享如何防止泄露私有客户端/域信息？
5. 语义原型为什么比传统原型更不模糊？
6. 风格仓库和调度机制的通信/存储/运行时开销？
7. 所有基线都匹配预算、使用本地BN、参数量相当时，收益还在吗？
8. 风格混合是否保持PACS/DomainNet上的标签？会不会产生off-manifold样本？

## Round 2: 作者回应 + 修改建议

### 更新评分：5/10（回应修正了framing但未解决实证风险）

### 最小实验集（从weak reject到borderline accept）

**主要benchmark**：
- PACS（必须）
- DomainNet子集（必须）
- Digit-5 / Office-Caltech10（支撑性）

**核心基线**：
- FedAvg, FedBN, FedProto
- FedSeProto 或最接近的解耦基线
- 一个风格共享但不解耦的基线（如FISC）

**关键消融**：
1. Full method
2. 解耦 + 无风格仓库
3. 风格仓库 + 不解耦
4. 随机调度 vs 按需调度
5. MSE对齐 vs 对比对齐
6. orth only / HSIC only / orth+HSIC

**解耦诊断**（比t-SNE重要得多）：
- z_sem 的类别预测准确率（应高）
- z_sem 的域预测准确率（应低 → 域无关）
- z_sty 的类别预测准确率（应低 → 不含语义）
- z_sty 的域预测准确率（应高 → 域特定）

**原型质量**：
- 类内紧凑度 / 类间分离度
- 与FedProto原型直接对比

**鲁棒性**：
- 各域单独准确率（非仅平均）
- 域间方差
- 至少3个seed

**开销**：
- 通信开销
- 服务器存储
- 训练时间 vs FedBN/FedProto

**可选高价值**：
- 风格原型的成员推理攻击

## 核心修改建议

### 必须添加
- 风格原型的精确定义（per-domain，非per-class）
- 解耦诊断实验（域/类预测准确率交叉分析）
- 原型紧凑度分析
- 匹配预算的消融实验
- 开销表格
- 局限性章节（不完美解耦 + 隐私风险）

### 必须删除
- "FIRST TIME" 声称
- 任何暗示"保证解纠缠或独立"的措辞
- 对先前工作的泛泛攻击（除非直接实验对比）
- 不必要的额外模块（考虑去掉L_sty_con如果消融证明非必要）

### 必须重新措辞

| 不要说 | 应该说 |
|--------|--------|
| "我们提出多个新模块" | "我们研究跨域FL中一种不同的共享原则：将域/风格因子作为增强资源共享，而非压制" |
| "风格是纯无关的可迁移信息" | "某些域变异在从语义中部分分离后，可被复用以提升鲁棒性" |
| "语义原型无模糊" | "解耦减少了域因子对语义的污染，实证提升了原型紧凑度" |

### 升级的概念钩子（GPT-5.4建议）

> ❌ 弱版本："风格有用，不应丢弃"
> 
> ✅ 强版本："在跨域FL中，并非所有全局有用信息都应作为模型参数聚合；某些信息应作为分解后的潜在资源共享。参数聚合是某些因子的错误共享机制，潜在风格迁移是一种替代通信原语，选择性因子共享可以超越压制或本地化。"
