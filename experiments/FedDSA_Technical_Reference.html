1.2 各模块维度

模块 输入维度 输出维度 参数量(约) 聚合策略
AlexNet Encoder [B, 3, 224, 224] [B, 1024] ~2.4M FedAvg (不含BN)

1.3 为什么这样设计聚合策略

聚合 (FedAvg):  Encoder卷积权重 + Semantic Head + Classifier
  → 这些是"所有域共享的语义理解能力"

私有不聚合:  BN 层 + Style Head
  → BN 层保留每个域自己的特征统计量 (FedBN 原则)
  → Style Head 保留每个域自己的风格投影方向

二、三大核心模块

2.1 Decouple (解耦)

目标: 让 z_sem 只包含语义信息, z_sty 只包含风格信息, 互不干扰。

正交约束 L_orth

z_sem_n = F.normalize(z_sem, dim=1)  # L2归一化
z_sty_n = F.normalize(z_sty, dim=1)  # L2归一化
cos = (z_sem_n * z_sty_n).sum(dim=1) # 逐样本余弦相似度 [B]
loss_orth = (cos ** 2).mean()         # 平方后取均值

公式:

L_orth = (1/B) * sum_i [ (cos(z_sem_i, z_sty_i))^2 ]

直觉: 如果 z_sem 和 z_sty 正交(余弦=0), loss=0。 如果它们有相关性(余弦!=0), loss > 0, 梯度会推它们变正交。 平方是因为我们不关心正相关还是负相关, 都要惩罚。

HSIC 核独立性约束 L_hsic

def _hsic(self, x, y):
    n = x.size(0)
    Kx = gaussian_kernel(x)  # 高斯核矩阵 [n, n]
    Ky = gaussian_kernel(y)  # 高斯核矩阵 [n, n]
    H = I - (1/n) * 11^T     # 中心化矩阵
    return trace(Kx @ H @ Ky @ H) / n^2

def gaussian_kernel(x):
    dist = ||x_i - x_j||^2        # 欧氏距离平方矩阵
    median = median(dist[dist>0])  # 中位数带宽
    bw = median / (2 * log(n+1))   # 自适应带宽
    return exp(-dist / (2 * bw))   # 高斯核

直觉: HSIC 测量两个变量的非线性依赖性。 正交只能捕捉线性相关, HSIC 能捕捉任何统计依赖。 当前实验中 lambda_hsic=0.0 (关闭), 仅用正交约束。

2.2 Share (风格共享)

目标: 用其他域的风格统计量做 AdaIN 增强, 让模型见到跨域变体。

风格仓库收集

# 每轮训练结束后, 客户端上传 h 空间的统计量
mu = style_sum / style_n           # h 在 batch 维度的均值 [1024]
var = style_sq_sum / style_n - mu^2  # 方差 = E[X^2] - (E[X])^2
sigma = sqrt(var.clamp(min=1e-6))    # 标准差 [1024]

# 服务器保存: style_bank[client_id] = (mu, sigma)

注意: 存的是 h (backbone 输出, 1024维), 不是 z_sty (style head 输出, 128维)。 因为 AdaIN 需要在 h 空间操作(去风格-加风格), 而 z_sty 空间太小且经过非线性变换。

AdaIN 风格增强 (原版 FedDSA)

def _style_augment(self, h):
    # 1. 从风格仓库随机取一个外域的 (mu, sigma)
    mu_ext, sigma_ext = random_pick(style_bank)  # [1024] each

    # 2. 计算本地 batch 的统计量
    mu_local = h.mean(dim=0)                      # [1024]
    sigma_local = h.std(dim=0).clamp(min=1e-6)    # [1024]

    # 3. 随机混合强度
    alpha = np.random.beta(0.1, 0.1)  # U形分布, 多数为0或1

    # 4. 混合统计量
    mu_mix = alpha * mu_local + (1-alpha) * mu_ext
    sigma_mix = alpha * sigma_local + (1-alpha) * sigma_ext

    # 5. AdaIN: 去掉本地风格, 穿上混合风格
    h_norm = (h - mu_local) / sigma_local    # 标准化 → 纯内容
    h_aug = h_norm * sigma_mix + mu_mix       # 上色 → 混合风格
    return h_aug  # [B, 1024]

流程图:

h [B,1024] ──→ 计算 mu_local, sigma_local
    |                    |
    v                    v
 标准化          风格仓库取 mu_ext, sigma_ext
 h_norm             |
    |            alpha 混合
    |               |
    v               v
 h_norm * sigma_mix + mu_mix = h_aug [B,1024]
    |
    v
 z_sem_aug = semantic_head(h_aug)
    |
    v
 loss_aug = CE(classifier(z_sem_aug), y)

2.3 Align (语义对齐)

目标: 拉近同类的 z_sem 和全局原型, 推开异类。

全局原型的收集

# 客户端: 在最后一个 epoch, 按类聚合 z_sem
for each sample (z_sem_i, y_i):
    proto_sum[y_i] += z_sem_i
    proto_count[y_i] += 1
local_proto[c] = proto_sum[c] / proto_count[c]  # 每类一个 [128] 向量

# 服务器: 加权平均所有客户端的同类原型
global_proto[c] = weighted_mean(all_clients_proto[c])

InfoNCE 对比损失

def _infonce_loss(self, z_sem, y):
    # proto_matrix: 所有类的全局原型 [C, 128], C=类别数(如7)
    proto_matrix = stack([global_proto[c] for c in classes])

    # L2 归一化
    z_n = normalize(z_sem)     # [B, 128]
    p_n = normalize(proto_matrix) # [C, 128]

    # 计算相似度矩阵 / 温度
    logits = (z_n @ p_n.T) / tau  # [B, C]

    # 交叉熵: 每个样本的正确类别应得到最高 logit
    loss = cross_entropy(logits, y)

公式:

L_InfoNCE = -(1/B) * sum_i [ log( exp(sim(z_i, p_yi)/tau) / sum_c exp(sim(z_i, p_c)/tau) ) ]

其中:
  sim(a, b) = (a/||a||) . (b/||b||)  (余弦相似度)
  p_yi = 样本 i 的正确类别的全局原型
  tau = 温度参数 (越小越尖锐)

tau 的作用: - tau=0.1: logits 被放大 10 倍, 分布尖锐 → 梯度集中在最难的样本 → 学习快但不稳定 - tau=0.2: logits 被放大 5 倍, 分布温和 → 梯度分散 → 学习慢但稳定 - tau=1.0: 原始余弦相似度, 几乎无学习信号

三、总损失函数

L_total = L_CE(z_sem, y)                        # 任务损失
        + L_CE(z_sem_aug, y)                     # 增强后任务损失
        + w * lambda_orth * L_orth               # 正交解耦
        + w * lambda_hsic * L_HSIC               # 核独立性
        + w * lambda_sem  * L_InfoNCE            # 语义对齐

其中 w = min(1.0, current_round / warmup_rounds)  # warmup 权重

损失项流向:

                        L_CE ←── z_sem ←── Semantic Head ←── h ←── Encoder ←── x
                                   ↑
                        L_CE ←── z_sem_aug ←── Semantic Head ←── h_aug (来自 AdaIN)
                                   |
                        L_orth ←── z_sem ↔ z_sty ──→ Style Head ←── h
                        L_hsic ←── z_sem ↔ z_sty
                                   |
                     L_InfoNCE ←── z_sem vs global_protos

四、Adaptive 改进 (feddsa_adaptive.py)

4.1 M1: 自适应增强强度

问题: 原版所有域用相同的 Beta(0.1,0.1) 随机 alpha, 不区分域差异。

改进: 用 z_sty 空间(纯风格)测量每个客户端的域偏移, 据此调整 alpha。

# --- 服务器端: 计算 gap ---
# 收集各客户端 z_sty 的 (mu, sigma) (不是 h 的!)
for each client i:
    raw_gap[i] = ||mu_zsty_i - mu_zsty_global||^2
               + ||sigma_zsty_i - sigma_zsty_global||^2

# EMA z-score 归一化
mean_gap, std_gap = EMA_update(mean_gap, std_gap, gaps)
z_score[i] = (raw_gap[i] - mean_gap) / std_gap
gap_normalized[i] = clip(z_score * 0.5 + 0.5, 0, 1)

# --- 客户端端: 自适应 alpha ---
alpha = aug_min + (aug_max - aug_min) * gap_normalized + N(0, noise_std)
alpha = clip(alpha, aug_min, aug_max)

# AdaIN 增强 (不同于原版!)
h_adain = h_norm * sigma_ext + mu_ext     # 完整风格迁移
h_aug = alpha * h_adain + (1-alpha) * h   # 按 alpha 比例混合

双 Bank 设计:

Bank 1: h-space (mu_h, sigma_h) [1024维] → 用于 AdaIN 增强
Bank 2: z_sty-space (mu_zsty, sigma_zsty) [128维] → 用于 gap 度量

为什么分开?
  h 适合做增强 (保留完整统计信息)
  z_sty 适合度量 gap (解耦后的纯风格信号更 discriminative)

4.2 M3: 域感知原型对齐

问题: 全局原型 = 所有域原型的均值 → 跨域平均导致语义稀释。

改进: 保留 (class, client_id) 独立原型, 用 SupCon InfoNCE。

# --- 服务器端: 存储域级原型 ---
domain_protos = {}
for client_id, protos in all_client_protos:
    for class_c, proto in protos:
        domain_protos[(class_c, client_id)] = proto
# 例: domain_protos = {
#   (dog, Photo): [128d], (dog, Art): [128d],
#   (dog, Cartoon): [128d], (dog, Sketch): [128d],
#   (cat, Photo): [128d], ... }

# --- 客户端端: SupCon Multi-Positive InfoNCE ---
proto_matrix = stack(all domain_protos)  # [N_total, 128]
proto_labels = [class for (class, cid) in keys]

z_n = normalize(z_sem)
p_n = normalize(proto_matrix)
logits = z_n @ p_n.T / tau  # [B, N_total]

# 对每个样本: 同类的所有域原型都是正样本
for sample i with label y_i:
    positives = {j : proto_labels[j] == y_i}  # 可能有 4 个 (4个域)
    loss_i = -(1/|P|) * sum_{p in positives} log(exp(logits[i,p]) / sum_j exp(logits[i,j]))

对比:

原版 InfoNCE:    每类 1 个正样本 (全局均值原型)
M3 SupCon InfoNCE: 每类 N 个正样本 (N=客户端数, 如4)

效果:
  原版: "你必须像平均狗" → 所有域的狗被拉到一个点 → 丢失域内结构
  M3:   "你只要是狗就行" → 允许不同域的狗在空间中有不同位置 → 更温和

4.3 实验模式

Mode 名称 M1 自适应 alpha M3 域感知原型 AdaIN 方式 InfoNCE 方式
0 Fixed Alpha 固定值(如0.5) 全局原型 h_adain blend 标准 InfoNCE
1 M1 Only gap → alpha 全局原型 h_adain blend 标准 InfoNCE
2 M3 Only Beta(0.1,0.1) 域级原型 h_adain blend SupCon InfoNCE

六、联邦学习通信流程

每轮 (Round) 的完整流程

Round t:

[Server]
  1. 打包下发: {全局模型参数, 全局原型, 风格仓库样本, round号, gap值}
     |
     v (下发到所有客户端)

[Client i] (并行)
  2. 接收并加载全局模型参数 (不含BN和Style Head)
  3. 本地训练 E 个 epoch:
     - 前向: x → h → z_sem, z_sty → output
     - 如果 round >= warmup:
       - 用风格仓库做 AdaIN 增强 → h_aug → z_sem_aug
     - 计算总损失 L_total
     - 反向传播 + 梯度裁剪 + SGD
  4. 收集本轮结果:
     - 更新后的模型参数
     - 按类聚合的语义原型 {class: z_sem均值}
     - h 空间的风格统计量 (mu_h, sigma_h)
     - z_sty 空间的风格统计量 (mu_zsty, sigma_zsty) [仅Adaptive]
     |
     v (上传到服务器)

[Server]
  5. 聚合模型参数: FedAvg (encoder + sem_head + classifier)
  6. 聚合语义原型: 加权平均 → 更新 global_protos
  7. 更新风格仓库: style_bank[cid] = (mu_h, sigma_h)
  8. [Adaptive] 计算 gap 指标: z_sty → raw_gap → EMA z-score → gap_normalized
  9. [Adaptive M3] 存储域级原型: domain_protos[(class, cid)] = proto

→ 进入 Round t+1
