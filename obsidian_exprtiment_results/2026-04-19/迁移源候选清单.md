# 候选 idea 榜(综合 brainstorm)

## 🥇 候选 1: FedDSA-ETF (Neural Collapse 分类头)

**一句话**: 把 `sem_classifier`(普通 Linear) 换成**固定的 Simplex ETF** — 让 L_orth 正交化的 z_sem 直接对齐 K 个理想单纯形顶点。

**迁移对**: A1 FedETF → Node 4

**为什么强**:
- 几何自洽: L_orth 保证 z_sem 内部正交 ⇒ ETF 要求 K 个类别 anchor 彼此等角 ⇒ 两者本质相同
- 零可训参数 ⇒ 无需聚合 ⇒ 差异化聚合里少一个噪声源
- 理论 backing: Neural Collapse (Papyan NAS'20), 证明最优分类器就是 ETF

**改造量**: ~15 行 (server init 构造 M, forward 替换 classifier)

**伪代码**:
```python
# serverdsa.__init__
K = num_classes; d = feat_dim
U = torch.linalg.qr(torch.randn(d, K))[0]  # [d, K] 正交
self.M = beta * sqrt(K/(K-1)) * U @ (I_K - 1_K 1_K^T / K)  # 固定,不训练

# clientdsa.forward (替代 self.sem_classifier)
z_sem_norm = F.normalize(z_sem, dim=-1)
logits = (z_sem_norm @ self.M) / tau
```

**预期**: PACS +0.5~1.5%, Office +0~0.5%, 稳定性显著提升(无分类器漂移)

**风险**: 低 (单变量替换,不破坏训练流程)

## 🥈 候选 2: FedDSA-TTA (SATA gate + T3A proto 纯推理端)

**一句话**: **训练不动**,只在 test 推理时,用 z_sty + style_bank 做 SATA 式风格交换 reliability gate + 用 z_sem 做 T3A 式在线 prototype 分类。

**迁移对**: B1 SATA + B2 T3A → Node 9

**为什么强**:
- **零训练成本** — 已经训好的 Plan A checkpoint 直接跑
- 免费利用 z_sty (之前只在训练时做正交约束,推理时扔掉,浪费)
- 免费利用 style_bank (同上)
- 完全可以叠加在**任何其他改进之上** (与候选 1/3 正交,可组合)

**改造量**: ~50 行纯推理代码 (不改 train.py)

**伪代码**:
```python
# clientdsa.test() 改造
@torch.no_grad()
def test_with_tta(x, model, style_bank, proto_bank):
    z_sem, z_sty = model.backbone_heads(x)
    logits = model.sem_classifier(z_sem)  # 原始
    H = entropy(softmax(logits))

    # SATA gate: style exchange 可靠性
    mu_ext, sig_ext = style_bank.sample_external(x.size(0))
    z_sem_swap = AdaIN(z_sem, mu_ext, sig_ext)
    logits_swap = model.sem_classifier(z_sem_swap)
    reliable = (logits.argmax(1) == logits_swap.argmax(1)) & (H < 0.4)

    # T3A proto: 可靠样本在线更新 class prototype
    for c in range(K):
        mask = reliable & (logits.argmax(1) == c)
        if mask.any():
            proto_bank[c] = 0.9 * proto_bank[c] + 0.1 * z_sem[mask].mean(0)

    # cosine 分类 + unreliable 回落到原 logits
    pred_proto = cosine_sim(z_sem, proto_bank).argmax(1)
    pred = torch.where(reliable, pred_proto, logits.argmax(1))
    return pred
```

**预期**: 所有 Plan A checkpoint 免费 +0.5~2% (无风险下限)

**风险**: 极低 (不改训练,如果不 work 直接回退到原 test)

## 🥉 候选 3: FedDSA-TIES (Sign-Conflict Aggregation)

**一句话**: 把 `aggregate_parameters()` 里的 FedAvg 换成 **TIES-Merging** (Trim + Sign Election + Disjoint Merge),PACS 4-outlier sign 冲突直接被消解。

**迁移对**: C1 TIES → Node 7

**为什么强**:
- PACS 4-outlier = Photo/Art/Cartoon/Sketch 4 个域梯度符号严重冲突 ⇒ FedAvg 抵消
- TIES 针对 "low-N multi-task" 场景设计,4-client 正是目标区间
- 非 FL 领域迁移 (符合用户"不一定是 FL 领域"的偏好)

**改造量**: ~30 行 server 代码 (替换 aggregate_parameters 核心循环)

**伪代码**:
```python
def ties_aggregate(global_state, client_states, density=0.2, lam=1.0):
    # 1. Trim: 每 client 保留 top-20% 大幅度更新
    task_vecs = [{k: c[k] - global_state[k] for k in c} for c in client_states]
    for tv in task_vecs:
        for k, v in tv.items():
            thresh = torch.quantile(v.abs().flatten(), 0.8)
            tv[k] = torch.where(v.abs() >= thresh, v, torch.zeros_like(v))
    # 2. Sign Election: 每参数选主导符号
    merged = {}
    for k in global_state:
        stacked = torch.stack([tv[k] for tv in task_vecs])
        elected_sign = torch.sign(stacked.sum(0))
        # 3. Disjoint Merge: 只平均同号项
        mask = (torch.sign(stacked) == elected_sign.unsqueeze(0)).float()
        merged[k] = global_state[k] + lam * (stacked * mask).sum(0) / mask.sum(0).clamp(min=1)
    return merged
```

**预期**: PACS +1~3%, Office 中性或 +0.5% (SAS 仍可作为 semantic_head 的 opt-in)

**风险**: 中 (需 ablation 验证 density/lam 超参,PACS 可能 sensitive)

## 候选 5: FedDSA-GeoConsist (几何一致性三层统一) ⚠️ 激进

**一句话**: ETF (训练几何) + L_orth (特征几何) + SATA+T3A (推理几何) — 把三个迁移源统一到**"Geometric Consistency across Training-Inference"** 叙事。

**迁移对**: A1 + B1 + B2 → Nodes 4 + 9

**叙事**:
- L_orth 让 z_sem ⊥ z_sty (特征空间几何)
- ETF 给 K 个类别提供等角顶点 (分类几何)
- T3A 在推理时把 test 样本 prototype 向 ETF 顶点聚拢 (推理几何)
- SATA 用 style_bank 做风格扰动可靠性检验

**改造量**: 大 (~150 行 + 需验证三者互不冲突)

**预期**: PACS +2~4%, Office +1~2% (强 story 加分)

**风险**: 中高 (三机制叠加需仔细消融,novelty check 可能有重叠)

## 选择理由

**进入 novelty-check 的 top-2**:

### Top-1: **FedDSA-ETF + TTA**(候选 1+2 组合 — "训练-推理几何双改造")
- 两个改动**互相正交,可分别/组合消融**
- 改造量都小 (训练端~15 行,推理端~50 行)
- 与 Plan A 完全兼容,不破坏正交解耦
- ETF 的 Neural Collapse backing + TTA 的 test-time 创新 = **双重 story**

### Top-2: **FedDSA-TIES**(候选 3 — "4-client Sign-Conflict Aggregation")
- 直接针对 PACS 4-outlier 痛点
- 迁移源是非 FL 领域的 model merging (符合用户"跨领域迁移"偏好)
- 单一机制替换,干净
- 可能替代 SAS (SAS 只在 Office 有效,TIES 双数据集通用)
