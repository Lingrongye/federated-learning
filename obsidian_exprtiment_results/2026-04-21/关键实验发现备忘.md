## 发现 1: CDANN 的 z_sty 类别泄漏在非线性 probe 下仍然严重

**时间**: 2026-04-21 凌晨

**数据 (MLP hidden=64 frozen probe, EXP-108 R200 checkpoint)**:

| Checkpoint | probe_sty_class linear | probe_sty_class MLP | Gap |
|-----------|----------------------|---------------------|-----|
| Office CDANN s=2 | 0.950 | 0.906 | -4.4pp |
| Office CDANN s=15 | 0.956 | 0.908 | -4.8pp |

**因果分析**:
- linear probe 95% 时,如果类别信息只是线性可读,MLP probe 应该掉很多 (因为 linear probe 已到饱和)。
- 实际 MLP 只比 linear 低 5pp 左右,说明 **z_sty 里的类别信息是非线性可读,不是单纯"线性结构性泄漏"**。
- Codex R2 review 预言此结论:"shared-trunk representations make loss-based branch purification a weak guarantee. CLUB reduces that concern; it does not remove it."

**对方案的意义**:
- CLUB (非线性 MI 上界) 的目的就是压非线性泄漏,但它只作用于 `z_sty` 分支,不动主干。
- 主干里类别信息全局可用 (因为 CE 在 z_sem 上训练),两个分支都从主干读出 → CLUB 只能压"读出"这一步,不能解决根因。
- **Stage-1 pilot 不启动**。改做验证性实验 (EXP-109/110/111) 证实结论稳健。

## 发现 3: Capacity probe (hidden sweep) 坐实 CLUB 无望

**时间**: 2026-04-21 凌晨 03:30

**6 个 EXP-108 checkpoint 全部跑完 capacity probe (hidden ∈ {16,64,128,256}):**

**PACS CDANN probe_sty_class**:
| seed | linear | MLP-16 | MLP-64 | MLP-128 | MLP-256 |
|------|--------|--------|--------|---------|---------|
| 2 | 0.963 | 0.959 | 0.953 | 0.962 | 0.962 |
| 15 | 0.960 | 0.949 | 0.957 | 0.949 | 0.954 |
| 333 | 0.963 | 0.911 | 0.964 | 0.962 | 0.965 |
| **mean** | **0.962** | **0.940** | **0.958** | **0.958** | **0.960** |

**Office CDANN probe_sty_class**:
| seed | linear | MLP-16 | MLP-64 | MLP-128 | MLP-256 |
|------|--------|--------|--------|---------|---------|
| 2 | 0.950 | 0.457† | 0.906 | 0.873 | 0.908 |
| 15 | 0.956 | 0.873 | 0.908 | 0.906 | 0.926 |
| 333 | 0.965 | 0.116† | 0.875 | 0.882 | 0.875 |
| **mean (m64+)** | **0.957** | - | **0.896** | **0.887** | **0.903** |

† MLP-16 早停不稳定,sklearn `early_stopping=True` 在少容量+小数据上偶发欠拟合。m64 起稳。

**关键结论**:
1. **PACS z_sty 对 class 的映射几乎"完全线性可读"** (linear 0.963, MLP 0.940-0.965 几乎一致)
   → 信息不是非线性藏起来的,是几乎平凡地可读
2. **Office 也一样线性为主,MLP 只比 linear 低 5pp**
   → 这 5pp 也只是 MLP 略欠拟合,不是"非线性深藏"
3. **probe 容量不是瓶颈**: hidden=16 已能读出大部分,hidden=256 饱和
4. **对 CLUB 的意义**: CLUB 目的是压非线性 MI 上界。但 z_sty 的 class 信息根本不是非线性的,CLUB 只能重复 linear GRL 做过的事,**无任何 structural gain**。

**Codex R2 "CLUB 不是结构性解药" 这个判断,现在有实证支撑:**
- Codex 的推理: 主干共享 → class 全局可读 → branch-level 损失无力
- 实证: z_sty probe_sty_class 几乎 = 主任务 accuracy (z_sem probe_sem_class ≈ 0.96) → **两个分支对类别的"读出能力"几乎相等**
- 说明双头并没有任何"信息分流",只是在读同一份主干的类别线索

**对下一步**:
- 等 EXP-109/110 的 orth_only 反事实 checkpoint 完成后 probe → 如果也是 0.94-0.96 → **CDANN 毫无贡献实锤**
- 等 EXP-111 的 lo=10 checkpoint 完成后 probe → 看加强正交能不能让数字明显下降

## 发现 4: 🔥 反事实 probe 颠覆 EXP-108 "C-probe anchor" 原结论

**时间**: 2026-04-21 早上 (EXP-109/110 第一批 4 个 checkpoint 完成后 capacity probe)

|-----|--------|--------|--------|---------|---------|
| CDANN s=2 | 0.963 | 0.959 | 0.953 | 0.962 | 0.962 |
| CDANN s=15 | 0.960 | 0.949 | 0.957 | 0.949 | 0.954 |
| CDANN s=333 | 0.963 | 0.911 | 0.964 | 0.962 | 0.965 |

| **Δ (CDANN - orth_only)** | **+72pp** | +67pp | +26pp | +25pp | +15pp |

### Office 上 CDANN vs orth_only (probe_sty_class)

| run | linear | MLP-16 | MLP-64 | MLP-128 | MLP-256 |
|-----|--------|--------|--------|---------|---------|
| CDANN 3-seed mean | 0.957 | — | 0.896 | 0.887 | 0.903 |
| orth_only s=2 | 0.958 | 0.954 | 0.947 | 0.954 | 0.956 |
| orth_only s=15 | 0.956 | 0.947 | 0.954 | 0.945 | 0.952 |
| orth_only s=333 | 0.972 | 0.963 | 0.954 | 0.965 | 0.969 |
| **orth_only 3-seed mean** | **0.962** | **0.955** | **0.952** | **0.955** | **0.959** |
| **Δ (CDANN - orth_only)** | **-0.005** | — | -0.056 | -0.068 | -0.056 |

### 🔥 颠覆性结论

1. **PACS 上 CDANN 确实"改变"了 z_sty 内容 (+72pp linear)**,但**方向错了**:
   - orth_only 的 z_sty 已经几乎不含 class (linear 0.240,接近 random 0.143)
   - CDANN 反而把 class 信息**错误地灌进**了 z_sty (linear 0.963)
   - 之前 EXP-108 的"anchor claim"说 CDANN "保留 class-relevant style" → **实际是 CDANN 的设计缺陷**:
     - GRL on z_sem 压 domain,非对抗 dom_head 监督 z_sty 学 domain → **encoder 被迫把 class 信号从 z_sem 路径挤出,漏到 z_sty**
     - 这破坏了双头解耦的初衷 (z_sty 应只含 style,不含 class)

2. **Office 上 CDANN 对 probe 几乎无差异** (两边都 ~0.96):
   - Office 风格弱 (4 domain 都是办公室物品照片),z_sty 自然就富含 class
   - CDANN 在 Office 上没能改变什么,"C-probe anchor" 在 Office 上本来就是空话

3. **EXP-108 原 "C-probe 铁证" 必须修正**:
   - 原结论: "CDANN 保留 class-relevant style (0.96 vs baseline 预期 0.15)"
   - 修正: **"CDANN 反向破坏了双头解耦,把 class 从 z_sem 挤到 z_sty"**
   - orth_only 才是真正的"clean decoupling" (z_sty probe 0.24)

4. **accuracy 也是 CDANN 输**:
   - EXP-108 CDANN PACS 80.08 (3-seed) vs EXP-109 orth_only s=2 = **82.23**
   - CDANN 的 GRL 机制**同时伤害了 z_sem 的 class 判别能力** → acc 掉 2pp
   - 等 EXP-109 s=15/333 完成才能算 3-seed mean

### 对方向的意义

- ❌ **CDANN 方向彻底死**: 不保留 anchor (虚假),accuracy 又掉,没理由继续
- ✅ **orth_only 才是 clean baseline**:
  - z_sty linear probe 0.24 = 真正的"解耦成功"
  - accuracy 82.23 > CDANN 80.08
- 🔍 **下一步**: 等 EXP-109 s=15/333 + EXP-111 lo=3/10 完成,看:
  - orth_only 3-seed 是否稳定在 82%+
  - 强正交 lo=10 能不能让 probe 进一步降到 ≤ 0.15 (random)

## 发现 6: 🔥 强正交 lo=3 是 probe-acc sweet spot, 但未能彻底"纯化"z_sty

**时间**: 2026-04-21 早上 (EXP-111 5 run 全完成 + capacity probe 全齐)

### 全 PACS 对照矩阵

| 方案 | λ_orth | CDANN | AVG Best | probe_sty_class (linear) | probe_sty_class (MLP-64) | probe_sty_class (MLP-256) |
|------|--------|-------|---------|-------------------------|-------------------------|--------------------------|
| CDANN (EXP-108) | 1 | ✓ | **80.08 ± 0.60** | 0.962 | 0.958 | 0.960 |
| orth_only (EXP-109) | 1 | ✗ | **80.64 ± 1.46** | **0.257** | 0.689 | 0.837 |
| strong_orth (EXP-111) | 3 | ✗ | 81.33 (s=2) | 0.339 | **0.201** 🏆 | 0.714 |
| strong_orth (EXP-111) | 10 | ✗ | 81.03 (s=2) | 0.280 | 0.339 | 0.799 |

### 三个关键发现

**1. CDANN vs orth_only 在 accuracy 上 *统计持平*** (80.08 vs 80.64, 在 std 内)
   - 之前 s=2 单 seed 看到 "orth_only 赢 1.4pp" 是噪声
   - CDANN 的 "+72pp linear probe" **没转化为 accuracy**
   - CDANN 改了 mechanism,没改 utility → Codex R2 的"branch-level loss 改不动 main task"实锤

**2. 强正交能大幅降低 z_sty 的非线性 class probe,但无法彻底删除**
   - lo=1 MLP-64 = 0.69 → lo=3 MLP-64 = **0.20** (-49pp,最干净)
   - 但 MLP-256 下: lo=3 仍 0.71 (-10pp),**class 信息只是被高阶非线性编码**
   - **正交约束压不掉 class 的"存在",只压它的"结构性出现"**

**3. 强正交单调轻伤 accuracy**
   - lo=1 (82.23 s=2) → lo=3 (81.33) → lo=10 (81.03): 单调降 ~0.5pp/5×
   - **sweet spot: lo=3** (probe 最好 + accuracy 损失最小)
   - lo=10 换来额外 probe 压制的收益 < accuracy 代价

### 对方向决策

**好消息**:
- 我们有 **clean decoupling mechanism** (lo=3 + orth_only),这是真正意义的"双头解耦"
- orth_only 在 PACS 上和 CDANN 持平 + Office 持平 → 不需要 CDANN 也能稳

**坏消息**:
- accuracy 天花板仍在 **~82%** (orth_only s=2) 或 **~80.6%** (3-seed mean)
- 无论 lo=1/3/10, accuracy 都在 80-82 之间波动,**光靠双头正交突破不了**
- CDANN 在 anchor 上失败 + accuracy 无增益 → **已验证过死路一条**

**需要思考的**:
- 要到 PACS >= 82+ (EXP-080 single 81.69 + 3-seed 80.64) 需要**新机制**,不是 probe 纯化
- 可能方向 (用户指令"不脱离双头解耦"):
  - **风格交换 (AdaIN / style swap)** — PARDON / CCST 思路, 在 clean z_sty 基础上做
  - **多原型语义 (FPL / FedPLVM)** — 保留 z_sem,在 server 上多原型
  - **正交 + 分类器正则化** — ETF anchoring 帮 z_sem 做 class 更稳

但这些都是未来方向,今晚验证性实验已经把"double head + CDANN 能做到什么" 答完了。

**判决**:

**对 paper 叙事的影响 (关键)**:
- 不能强行卖"VIB 通过更好解耦涨点" → reviewer 做 probe 会立刻打脸
- 诚实叙事: "VIB 通过信息瓶颈正则化 + 原型锚定在 FL 跨域上涨点, probe 揭示这不是传统意义的解耦"
- 同时注意: **orth_uc1 本身 Office probe_sty_class 96%** — 所有方法都没做到信息论解耦, 这是共同短板
