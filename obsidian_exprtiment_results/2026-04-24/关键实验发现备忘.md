# 2026-04-24 关键实验发现备忘

## 发现 1: EXP-125 OCSD 验证 1 推翻原假设 (2026-04-24 早晨)

**时间**: 2026-04-24 ~04:00

**背景**: OCSD 方向提出 "PACS Art 的 13-15% over_conf_wrong 是 style shortcut 样本". 要验证生死线, 写 inference script 从 sgpa_vib s=2 checkpoint 提取 Art test set 的 over_conf_wrong 样本, 人眼看 24 张图 (Art test split 只 204 样本, over_conf_wrong 24 个).

**数据**: Art test 24 个 over_conf_wrong 样本 (confidence > 0.8 且预测错)

**用户人眼观察结论** (关键):

> "感觉就是完全学错了. 比如长颈鹿跟马可能相似, **其他都是其实还是能认得出来的 但是模型理解错了**"

### 含义分析

**不是 A 类 (风格极端 shortcut)**: 图**不特别抽象**, 是普通 Art 绘画, 人眼能认出

**不是 B 类 (本质难/标注歧义)**: 图本身信息足够, 不是难样本

**是 D 类 (新类别)**: **model 的 class boundary 学歪**, 不是图的问题, 也不是 style 偷懒

### 机制推测

问题不是 "model 偷看 style → class", 而是:
- FL 跨域训练, **Art 的 class 表征没有与其他 domain 的 class 表征对齐**
- classifier 边界基于 mixed features 切分, 但 Art-dog / Art-horse / Art-person 在 feature space 里的位置系统性偏离
- 测试时 Art person 的特征正好落在 Art dog 错位的 cluster 里 → 错预测

### 对方向选择的含义

**推翻**: OCSD 单独方向 (MixStyle 扰动检测 style shortcut) — D 类不是 style 偷懒, MixStyle 扰动机制不精准

**支持**: **CDCA-Orth** (跨域 class prototype anchor) — 直接对齐 Art-dog 到 Photo/Cartoon/Sketch-dog, 修正边界错位

**新方向推荐**:
- 废掉 OCSD (原计划的 5 步验证 + paper claim)
- 走 **CDCA-Orth** (cross-domain class prototype anchor with gap-aware gating)
- 理论 claim: "跨域 FL 中, class 表征在各 domain 独立漂移, 需要 explicit class-level alignment 而非 style-level shortcut fix"

### 下一步

1. 放弃 OCSD 方向 (不走 MixStyle 扰动 + selective invariance)
2. 设计 **CDCA-Orth 方法** 并实施 (EXP-126 或新 EXP-127)
3. 等待 orth_only s=2 R=200 with save_model 完成 (明早 ~7 AM), 用更严格的 orth 模型再验证 D 类观察 (checkpoint 不匹配 sgpa_vib 的 feature space, 但 D 类是跨 model 的 phenomenon)

### 数据证据

- Art test split: 204 samples
- Over-conf wrong: 24 (11.8%, 与 Stage B 13-15% 一致)
- Confusion top: person→dog (5), dog→elephant (3), guitar→person (2), horse→elephant (2)
- **错法分散**, 不集中在某类 (和 Stage B 的 "Art-guitar +24 FDSE win" 不同)
- 这也支持 D 类: 不是 single hard cell, 是 general class boundary 错位
