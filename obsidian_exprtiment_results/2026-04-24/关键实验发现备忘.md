# 2026-04-24 关键实验发现备忘

## 发现 1: EXP-125 OCSD 验证 1 推翻原假设 (2026-04-24 早晨)

**时间**: 2026-04-24 ~04:00

**背景**: OCSD 方向提出 "PACS Art 的 13-15% over_conf_wrong 是 style shortcut 样本". 要验证生死线, 写 inference script 从 sgpa_vib s=2 checkpoint 提取 Art test set 的 over_conf_wrong 样本, 人眼看 24 张图 (Art test split 只 204 样本, over_conf_wrong 24 个).

**数据**: Art test 24 个 over_conf_wrong 样本 (confidence > 0.8 且预测错)

**用户人眼观察结论** (关键):

> "感觉就是完全学错了. 比如长颈鹿跟马可能相似, **其他都是其实还是能认得出来的 但是模型理解错了**"

### 含义分析

**不是 A 类 (风格极端 shortcut)**: 图**不特别抽象**, 是普通 Art 绘画, 人眼能认出

**不是 B 类 (本质难/标注歧义)**: 图本身信息足够, 不是难样本

**是 D 类 (新类别)**: **model 的 class boundary 学歪**, 不是图的问题, 也不是 style 偷懒

### 机制推测

问题不是 "model 偷看 style → class", 而是:
- FL 跨域训练, **Art 的 class 表征没有与其他 domain 的 class 表征对齐**
- classifier 边界基于 mixed features 切分, 但 Art-dog / Art-horse / Art-person 在 feature space 里的位置系统性偏离
- 测试时 Art person 的特征正好落在 Art dog 错位的 cluster 里 → 错预测

### 对方向选择的含义

**推翻**: OCSD 单独方向 (MixStyle 扰动检测 style shortcut) — D 类不是 style 偷懒, MixStyle 扰动机制不精准

**支持**: **CDCA-Orth** (跨域 class prototype anchor) — 直接对齐 Art-dog 到 Photo/Cartoon/Sketch-dog, 修正边界错位

**新方向推荐**:
- 废掉 OCSD (原计划的 5 步验证 + paper claim)
- 走 **CDCA-Orth** (cross-domain class prototype anchor with gap-aware gating)
- 理论 claim: "跨域 FL 中, class 表征在各 domain 独立漂移, 需要 explicit class-level alignment 而非 style-level shortcut fix"

### 下一步

1. 放弃 OCSD 方向 (不走 MixStyle 扰动 + selective invariance)
2. 设计 **CDCA-Orth 方法** 并实施 (EXP-126 或新 EXP-127)
3. 等待 orth_only s=2 R=200 with save_model 完成 (明早 ~7 AM), 用更严格的 orth 模型再验证 D 类观察 (checkpoint 不匹配 sgpa_vib 的 feature space, 但 D 类是跨 model 的 phenomenon)

### 数据证据

- Art test split: 204 samples
- Over-conf wrong: 24 (11.8%, 与 Stage B 13-15% 一致)
- Confusion top: person→dog (5), dog→elephant (3), guitar→person (2), horse→elephant (2)
- **错法分散**, 不集中在某类 (和 Stage B 的 "Art-guitar +24 FDSE win" 不同)
- 这也支持 D 类: 不是 single hard cell, 是 general class boundary 错位

## 发现 2: Decouple Probe 颠覆 "共享 trunk 诅咒" 假说 (2026-04-24 下午)

**时间**: 2026-04-24 ~05:00

**背景**: Codex R2 诊断 "shared-trunk 诅咒" 假说 — z_sty 的 class 泄漏因为两路读同一批 trunk channel. 为验证,写 visualize_decouple.py 直接提取 semantic_head.0.weight 和 style_head.0.weight ([128, 1024]), 计算 3 个指标在 3 个 ckpt 上可视化.

**3 个 ckpt (PACS s=2/s=15 VIB + Office s=15 plain SGPA) 结果一致**:

| 指标 | 值 | 理想 | 诅咒信号 |
|---|:-:|:-:|:-:|
| W_sem·W_sty^T 行 cos 绝对均值 | **0.025** | 0 | >0.1 |
| W_sem·W_sty^T 行 cos 最大 | 0.12 | 0 | >0.5 |
| channel usage pearson | **0.04 / -0.011 / -0.016** | 0 | >0.5 |
| heavy-overlap (top-25% 重合) | 64/256 (25%) | <64 | >128 |

### 颠覆性结论

**L_orth 在权重 + channel 层面近乎完美**:
- 128 sem 行 × 128 sty 行的 16384 对 cos 平均 0.025 (几乎全白 heatmap)
- 两路对 1024 trunk channel 的依赖 **pearson ≈ 0, 完全独立**
- 重度 channel 重合率 = 25% = 完美随机

**但 EXP-111 probe_sty_class MLP-256 = 0.81** 泄漏依然在.

### 机制修正

**原诅咒假说 (Codex R2)**: 两路共享 trunk channel → class 重复读出
**新机制**: L_CE 让 trunk **每个 channel 都含 class 信息** (pooled 所有维度都被 CE 梯度影响过). 无论 z_sty 挑哪批 channel, 每个 channel 里都有 class 残余.

**病根不在 head 层的共享, 在 trunk 本身的 class-wide dispersion**.

### 对三选项判决

| 选项 | 判决 | 理由 |
|:-:|:-:|---|
| A 权重行正交 | **完全无效** | L_orth 已把 corr 压到 0.025 物理下限 |
| B channel mask 互斥 | **完全无效** | channel 使用已经自然独立了 |
| C spatial mask (F2DC) | 唯一 head 路线 | 换维度 (空间 vs channel), 空间上 class 不是全局弥散 |

### 新路径建议

**绕开 head 战场**:
- 跨域 class prototype anchor (不动 trunk, 直接在 z_sem 上做跨 domain 拉齐)
- EXP-125 OCSD 方向已因 D 类样本推翻, **CDCA-Orth 成为主要候选**

## 发现 3: PGA (Prototype-Guided Attribution) 方法论死亡 (2026-04-24 晚)

**时间**: 2026-04-24 ~07:00~08:00

**背景**: 探索 feature-map 级 attribution, 用跨 client 聚合的 class 原型对每个 (h,w) 位置的 feature 打 cos 相似度分. 设计 3 版:
- v1: global-pooled prototype + 单一 encoder — Cartoon 完全错位
- v2: per-location prototype + per-domain encoder — Sketch/Photo 反 (红在背景蓝在主体)
- v3: preprocess 对齐训练 (256×256 + /255, 去掉 ImageNet norm) — **Cartoon 命中, Photo/Sketch 仍反**

### v3 结果 (决定性)

| Domain | 主体位置 | α 落在哪 | 判决 |
|---|---|---|:-:|
| Art | 中(木吉他) | 中+背景 | 🟡 部分 |
| Cartoon | 垂直中央 | 垂直红色带 | ✅ |
| Photo | 中(黑电吉他) | 红在背景,蓝在主体 | ❌ 反 |
| Sketch | 中央线条 | 红散在背景,蓝在吉他区 | ❌ 反 |

### 数学病根因

cos-with-mean-prototype 的固有问题:
- 非负 feature (ReLU 后) + cos 天然下限高 (0.86+) → 信号 range 窄
- **平均操作强化"多样本一致部分"(背景), 抹平"多样本差异大部分"(主体)**
- 结果: 背景跟平均对得上 (cos 高), 主体跟平均对不上 (cos 低)

### 合照类比

100 张学生合照叠起来求平均:
- 墙/桌椅 (每张都一样) → 平均后清晰
- 学生 (每张姿势不同) → 平均后糊成一团

新照片跟平均比:
- 墙位置 → cos 高 → 红
- 学生位置 → cos 低 → 蓝

完全对应 PGA 的 Photo/Sketch 反向现象.

### 用户关键洞察

> "用像素去逼近原型本来就不可取 — 应该像 F2DC 那样**训练一个东西**, 让这些特征能显性地靠真正的判别特征去评判哪些是判别主体, 而不是因为位置改变而改变"

**这句话精准命中**: static pixel-to-prototype matching 是位置依赖的; learned attribution (F2DC DFD 那种) 才是位置不变的判别.

### PGA 方向正式关闭

- 要救就得加 gradient → 等于 Grad-CAM → novelty 归零
- 跨 domain 平均 prototype 在 Sketch/Photo 根本不指向主体
- 继续修 = 重新发明 Grad-CAM

## 发现 4: 兜底方案 FedDSA-Calibrator 已成文 (2026-04-24 晚)

**时间**: 2026-04-24 ~08:30

**背景**: 所有 head 层面探索(CDANN, 强 L_orth, VIB, PGA, decouple_probe 选项 A/B) 都已证明失败或无效. 用户要求写兜底文档.

### 方案要点

- **位置**: pooled 级 (不需要 spatial mask, 不需要 feature-map 级改动)
- **核心**: 加 corrector (50 行 MLP) 从 z_sty 里显式提取 class 信号
- **Loss**: L_CE + L_orth + λ_aux · CE(aux_cls(z_sty_cal), y)
- **Final**: classifier(z_sem + λ · z_sty_cal)
- **Warmup**: R0-20 无 calibrator, R20-40 ramp, R40+ 稳态

### 预期

- Target: Office +1.5~2.5 pp (填 -1.49 差距)
- Conservative: Office +0.4~0.9 pp (至少不退)
- Failure: accuracy 持平或略退, 需 kill 重来

### 定位

**兜底, 不是主攻**:
1. 首选: 跨域 prototype anchor (CDCA-Orth) — 不碰 z_sty
2. 次选: F2DC 选项 C spatial DFD — 200 行, 3 新超参
3. **兜底: FedDSA-Calibrator** — 50 行, 1 新超参, 保证不退

## 发现 5: 兜底方案修正 — 真正要的是 DualEncoder 物理解耦 (2026-04-24 晚)

**时间**: 2026-04-24 ~09:00

**背景**: 用户纠正 — 之前说的兜底不是 Calibrator (pooled 级加 MLP), 而是**独立双 encoder** 的方案 — "走双通道训练, 会导致参数量变大那个线路".

### 两个兜底方案本质差异

| 维度 | Calibrator (兜底 A) | **DualEncoder (兜底 B, 用户真正要的)** |
|---|---|---|
| 哲学 | 接受泄漏, 利用它 | **从源头杜绝泄漏** |
| 架构 | pooled 级加 MLP | **双完整 AlexNet encoder** |
| 参数量 | 1.01× | **2×** |
| 训练时间 | 1.05× | 1.5× |
| probe_sty_class 下降 | 否 | **是 (0.81 → <0.3)** ⭐ |
| Novelty | 中 | **高** (FL 里独立双 trunk 少见) |

### DualEncoder 关键设计

**物理保证 encoder_sty 的 trunk 不被 L_CE 梯度影响**:
- L_CE 只反传到 encoder_sem
- encoder_sty 只被 L_orth + 可选 L_sty_aux 训练
- trunk 每个 channel 都不含 class 梯度历史 → class mirror 泄漏**在源头就不会发生**

**FL 聚合**: encoder_sty 本地私有 (选项 Y), 通信量回到 1×, 符合 "style 本地" 先验.

**防坍缩**: L_sty_aux = CE(dom_head(z_sty), domain_id), 便宜 (9K params), **不走 CDANN 的 GRL 老路** (独立 encoder 不担心 class 被挤过去).

### 方案文档

- 位置: experiments/FALLBACK_PLAN_DualEncoder.md
- 覆盖: 13 节, 3 个变体 (A 完全独立 / B 前共享后分叉 / C adapter) + FL 聚合策略 (X/Y/Z) + loss + 代码骨架 + 预期 + 风险 + 决策定位

### 预期

- Target: Office +0.9~2.9 pp, PACS +0.9~1.9 pp
- 保底: 即使 accuracy 持平, probe_sty_class 暴跌 (0.81 → <0.3) 就是 paper 级 representation 证据

### 兜底方案新排序

1. **DualEncoder (兜底 B, 首选)**: 真·物理解耦, probe 证据 + accuracy 双收
2. Calibrator (兜底 A, 次选): 代价最小, 但不治本, 只有 DualEncoder 也失败时才启动

### 何时启动

- 首选路线 (跨域 prototype anchor / CDCA-Orth) 失败
- 次选路线 (F2DC 选项 C spatial DFD) 失败或太贵
- **此时启动 DualEncoder**, 不先试 Calibrator
