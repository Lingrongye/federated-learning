## 发现 1: EXP-125 OCSD 验证 1 推翻原假设 (2026-04-24 早晨)

**时间**: 2026-04-24 ~04:00

**背景**: OCSD 方向提出 "PACS Art 的 13-15% over_conf_wrong 是 style shortcut 样本". 要验证生死线, 写 inference script 从 sgpa_vib s=2 checkpoint 提取 Art test set 的 over_conf_wrong 样本, 人眼看 24 张图 (Art test split 只 204 样本, over_conf_wrong 24 个).

**数据**: Art test 24 个 over_conf_wrong 样本 (confidence > 0.8 且预测错)

**用户人眼观察结论** (关键):

> "感觉就是完全学错了. 比如长颈鹿跟马可能相似, **其他都是其实还是能认得出来的 但是模型理解错了**"

### 含义分析

**不是 A 类 (风格极端 shortcut)**: 图**不特别抽象**, 是普通 Art 绘画, 人眼能认出

**不是 B 类 (本质难/标注歧义)**: 图本身信息足够, 不是难样本

**是 D 类 (新类别)**: **model 的 class boundary 学歪**, 不是图的问题, 也不是 style 偷懒

### 机制推测

问题不是 "model 偷看 style → class", 而是:
- FL 跨域训练, **Art 的 class 表征没有与其他 domain 的 class 表征对齐**
- classifier 边界基于 mixed features 切分, 但 Art-dog / Art-horse / Art-person 在 feature space 里的位置系统性偏离
- 测试时 Art person 的特征正好落在 Art dog 错位的 cluster 里 → 错预测

### 对方向选择的含义

**推翻**: OCSD 单独方向 (MixStyle 扰动检测 style shortcut) — D 类不是 style 偷懒, MixStyle 扰动机制不精准

**支持**: **CDCA-Orth** (跨域 class prototype anchor) — 直接对齐 Art-dog 到 Photo/Cartoon/Sketch-dog, 修正边界错位

**新方向推荐**:
- 废掉 OCSD (原计划的 5 步验证 + paper claim)
- 走 **CDCA-Orth** (cross-domain class prototype anchor with gap-aware gating)
- 理论 claim: "跨域 FL 中, class 表征在各 domain 独立漂移, 需要 explicit class-level alignment 而非 style-level shortcut fix"

### 下一步

1. 放弃 OCSD 方向 (不走 MixStyle 扰动 + selective invariance)
2. 设计 **CDCA-Orth 方法** 并实施 (EXP-126 或新 EXP-127)
3. 等待 orth_only s=2 R=200 with save_model 完成 (明早 ~7 AM), 用更严格的 orth 模型再验证 D 类观察 (checkpoint 不匹配 sgpa_vib 的 feature space, 但 D 类是跨 model 的 phenomenon)

### 数据证据

- Art test split: 204 samples
- Over-conf wrong: 24 (11.8%, 与 Stage B 13-15% 一致)
- Confusion top: person→dog (5), dog→elephant (3), guitar→person (2), horse→elephant (2)
- **错法分散**, 不集中在某类 (和 Stage B 的 "Art-guitar +24 FDSE win" 不同)
- 这也支持 D 类: 不是 single hard cell, 是 general class boundary 错位

## 发现 2: Decouple Probe 颠覆 "共享 trunk 诅咒" 假说 (2026-04-24 下午)

**时间**: 2026-04-24 ~05:00

**背景**: Codex R2 诊断 "shared-trunk 诅咒" 假说 — z_sty 的 class 泄漏因为两路读同一批 trunk channel. 为验证,写 visualize_decouple.py 直接提取 semantic_head.0.weight 和 style_head.0.weight ([128, 1024]), 计算 3 个指标在 3 个 ckpt 上可视化.

**3 个 ckpt (PACS s=2/s=15 VIB + Office s=15 plain SGPA) 结果一致**:

| 指标 | 值 | 理想 | 诅咒信号 |
|---|:-:|:-:|:-:|
| W_sem·W_sty^T 行 cos 绝对均值 | **0.025** | 0 | >0.1 |
| W_sem·W_sty^T 行 cos 最大 | 0.12 | 0 | >0.5 |
| channel usage pearson | **0.04 / -0.011 / -0.016** | 0 | >0.5 |
| heavy-overlap (top-25% 重合) | 64/256 (25%) | <64 | >128 |

### 颠覆性结论

**L_orth 在权重 + channel 层面近乎完美**:
- 128 sem 行 × 128 sty 行的 16384 对 cos 平均 0.025 (几乎全白 heatmap)
- 两路对 1024 trunk channel 的依赖 **pearson ≈ 0, 完全独立**
- 重度 channel 重合率 = 25% = 完美随机

**但 EXP-111 probe_sty_class MLP-256 = 0.81** 泄漏依然在.

### 机制修正

**原诅咒假说 (Codex R2)**: 两路共享 trunk channel → class 重复读出
**新机制**: L_CE 让 trunk **每个 channel 都含 class 信息** (pooled 所有维度都被 CE 梯度影响过). 无论 z_sty 挑哪批 channel, 每个 channel 里都有 class 残余.

**病根不在 head 层的共享, 在 trunk 本身的 class-wide dispersion**.

### 对三选项判决

| 选项 | 判决 | 理由 |
|:-:|:-:|---|
| A 权重行正交 | **完全无效** | L_orth 已把 corr 压到 0.025 物理下限 |
| B channel mask 互斥 | **完全无效** | channel 使用已经自然独立了 |
| C spatial mask (F2DC) | 唯一 head 路线 | 换维度 (空间 vs channel), 空间上 class 不是全局弥散 |

### 新路径建议

**绕开 head 战场**:
- 跨域 class prototype anchor (不动 trunk, 直接在 z_sem 上做跨 domain 拉齐)
- EXP-125 OCSD 方向已因 D 类样本推翻, **CDCA-Orth 成为主要候选**

- NOTE: experiments/ablation/decouple_probe/NOTE.md

## 发现 3: PGA (Prototype-Guided Attribution) 方法论死亡 (2026-04-24 晚)

**时间**: 2026-04-24 ~07:00~08:00

**背景**: 探索 feature-map 级 attribution, 用跨 client 聚合的 class 原型对每个 (h,w) 位置的 feature 打 cos 相似度分. 设计 3 版:
- v1: global-pooled prototype + 单一 encoder — Cartoon 完全错位
- v2: per-location prototype + per-domain encoder — Sketch/Photo 反 (红在背景蓝在主体)
- v3: preprocess 对齐训练 (256×256 + /255, 去掉 ImageNet norm) — **Cartoon 命中, Photo/Sketch 仍反**

### v3 结果 (决定性)

| Domain | 主体位置 | α 落在哪 | 判决 |
|---|---|---|:-:|
| Art | 中(木吉他) | 中+背景 | 🟡 部分 |
| Cartoon | 垂直中央 | 垂直红色带 | ✅ |
| Photo | 中(黑电吉他) | 红在背景,蓝在主体 | ❌ 反 |
| Sketch | 中央线条 | 红散在背景,蓝在吉他区 | ❌ 反 |

### 数学病根因

cos-with-mean-prototype 的固有问题:
- 非负 feature (ReLU 后) + cos 天然下限高 (0.86+) → 信号 range 窄
- **平均操作强化"多样本一致部分"(背景), 抹平"多样本差异大部分"(主体)**
- 结果: 背景跟平均对得上 (cos 高), 主体跟平均对不上 (cos 低)

### 合照类比

100 张学生合照叠起来求平均:
- 墙/桌椅 (每张都一样) → 平均后清晰
- 学生 (每张姿势不同) → 平均后糊成一团

新照片跟平均比:
- 墙位置 → cos 高 → 红
- 学生位置 → cos 低 → 蓝

完全对应 PGA 的 Photo/Sketch 反向现象.

### 用户关键洞察

> "用像素去逼近原型本来就不可取 — 应该像 F2DC 那样**训练一个东西**, 让这些特征能显性地靠真正的判别特征去评判哪些是判别主体, 而不是因为位置改变而改变"

**这句话精准命中**: static pixel-to-prototype matching 是位置依赖的; learned attribution (F2DC DFD 那种) 才是位置不变的判别.

### PGA 方向正式关闭

- 要救就得加 gradient → 等于 Grad-CAM → novelty 归零
- 跨 domain 平均 prototype 在 Sketch/Photo 根本不指向主体
- 继续修 = 重新发明 Grad-CAM

## 发现 5: 兜底方案修正 — 真正要的是 DualEncoder 物理解耦 (2026-04-24 晚)

**时间**: 2026-04-24 ~09:00

**背景**: 用户纠正 — 之前说的兜底不是 Calibrator (pooled 级加 MLP), 而是**独立双 encoder** 的方案 — "走双通道训练, 会导致参数量变大那个线路".

### 两个兜底方案本质差异

| 维度 | Calibrator (兜底 A) | **DualEncoder (兜底 B, 用户真正要的)** |
|---|---|---|
| 哲学 | 接受泄漏, 利用它 | **从源头杜绝泄漏** |
| 架构 | pooled 级加 MLP | **双完整 AlexNet encoder** |
| 参数量 | 1.01× | **2×** |
| 训练时间 | 1.05× | 1.5× |
| probe_sty_class 下降 | 否 | **是 (0.81 → <0.3)** ⭐ |
| Novelty | 中 | **高** (FL 里独立双 trunk 少见) |

### DualEncoder 关键设计

**物理保证 encoder_sty 的 trunk 不被 L_CE 梯度影响**:
- L_CE 只反传到 encoder_sem
- encoder_sty 只被 L_orth + 可选 L_sty_aux 训练
- trunk 每个 channel 都不含 class 梯度历史 → class mirror 泄漏**在源头就不会发生**

**FL 聚合**: encoder_sty 本地私有 (选项 Y), 通信量回到 1×, 符合 "style 本地" 先验.

**防坍缩**: L_sty_aux = CE(dom_head(z_sty), domain_id), 便宜 (9K params), **不走 CDANN 的 GRL 老路** (独立 encoder 不担心 class 被挤过去).

- 位置: experiments/FALLBACK_PLAN_DualEncoder.md
- 覆盖: 13 节, 3 个变体 (A 完全独立 / B 前共享后分叉 / C adapter) + FL 聚合策略 (X/Y/Z) + loss + 代码骨架 + 预期 + 风险 + 决策定位

## 发现 6: FedShapeGate 假设证伪 — AlexNet on PACS 全 channel 都是 texture-biased (2026-04-24 深夜)

**时间**: 2026-04-24 ~10:00 (经过 6 轮 codex review 后跑出干净结果)

**背景**: 用户提议从 F2DC 启发想新方案 — feature map 级 channel-wise 解耦, 用"shape sensitivity vs texture sensitivity"作为可测量物理判据筛选 class channel. 跑 sanity probe 验证假设是否成立.

### Probe 设计 (经过 codex 6 轮 review)

- 输入: 32 张 PACS 图 (8/domain, 7 class 各一)
- 扰动: ElasticTransform (变形保纹理) + FDA-style 低频窗口 amp swap (变纹理保形体)
- 度量: per-channel `pooled sum(diff)/sum(norm)`, 排除低 energy 10% channel
- Bootstrap CI: stratified by cid + nested donor, mirror point estimator

### 核心结果 (PACS s=2 VIB ckpt)

```
pearson(shape_sens, tex_sens) = 0.547 [95% CI: 0.453, 0.571]
shape_sens range [0.093, 0.180]    ← 形体扰动反应极弱
tex_sens   range [0.370, 0.693]    ← 纹理扰动反应是形体的 2-4×

#channels shape > tex = 0 / 230    ⭐ 关键!
#channels tex > shape = 230 / 230
```

### 决定性 finding

**230 / 230 个有效 channel, 所有 tex_sens 都大于 shape_sens.** 整个云团完全在 y=x 对角线上方, 没有任何 shape-dominant channel 可分给 z_sem.

### 含义

**FedShapeGate 假设直接失败**:
- 原假设: 存在 shape-dominant channel 和 texture-dominant channel 可 gate 分离
- 实测: AlexNet on PACS 整个 conv5 都是 texture-biased, 没有 shape-dominant region
- → 即使做完美 gate 也是空 gate (所有 channel 都被门关向 z_sty, 等于没做)

### 但 finding 本身有价值

1. **印证 Geirhos 2019**: ImageNet-trained CNN 是 texture-biased — 这在 PACS-trained 上仍然成立
2. **解释跨 domain 泛化差的真正根因**: domain 间差异主要是 texture, texture-biased encoder 跨 domain 必然崩
3. **指向新方向**: 不是事后 gate, 是**训练时就让 encoder 学 shape-bias** (SIN-style stylized training)

### 对总体策略的影响

**FedShapeGate 路线作废**, 但收获了:
- 一个 paper 级 motivation figure (230/230 cluster 在 y=x 上方)
- 明确的下一步指向: shape-bias training 而非 channel gate
- 一个经过 codex 6 轮 review 的可信 probe 工具 (复用价值高)

### 文件位置

- Probe 脚本: FDSE_CVPR25/scripts/shape_tex_sens_probe.py
- 散点图: experiments/ablation/shape_tex_probe/figs/shape_tex_sens_scatter.png
- 原始数据: experiments/ablation/shape_tex_probe/figs/shape_tex_sens_data.npz
- Codex review 流程: 6 轮 BLOCK→NEEDS_FIXES→APPROVE 的迭代证据 (Round 1-6)

### 下一步建议

1. 不再走 channel-level gating 方向
2. 兜底方案排序保持: DualEncoder (B, 首选) > Calibrator (A, 次选)
3. 主攻方向: 跨域 class prototype anchor (CDCA-Orth)
4. 长期方向 (新增): 探索 SIN-style stylized training 让 encoder 学 shape-bias

## 发现 N+1: 兜底方案 DualEncoder 经 research-refine 4 轮升级为 BiProto (2026-04-24, 8.75/10)

**时间**: 2026-04-24
**背景**: 原 DualEncoder 方案 (2× AlexNet 独立双 trunk) 经用户 + GPT-5.4 4 轮 review 大幅修订

**最终方案 BiProto**: 完整见 `refine_logs/2026-04-24_feddsa-biproto-asym_v1/FINAL_PROPOSAL.md`, 大白话 `知识笔记/大白话_FedDSA-BiProto方案.md`.

**4 轮演化 (6.5 → 7.8 → 8.25 → 8.75)**:

| 版本 | 核心修改 | 为什么改 |
|---|---|---|
| v1 → v2 | 删 L_sem_proto; Pd 改 domain-indexed; 非对称 encoder 降为 enabling infra; C0 改 matched intervention | **Contribution sprawl**: 原版说双原型但实际是 2 条 loss, 稀释焦点; **叙事矛盾**: 说修 trunk 但 taps.detach() 根本没动 trunk |
| v2 → v3 | Straight-through 混合轴 (forward=Pd, grad=batch centroid via ST); C0 只冻 encoder_sem (head 可训); Pc 完全降级为 monitor | **Headline/impl 错配**: 说用 federated Pd 实际用 batch local; **C0 冻死 prediction path** 测不到增量 |
| v3 → v4 | ST normalization 明确 (final F.normalize); L_proto_excl pilot 只用 present_classes; "to our knowledge" + Limitations §; -Pd MANDATORY | **D=K=4 empirical 弱**: Pd 和 per-client bank 实现等价, 必须 -Pd ablation 实证 |

**关键技术决策 (跨 4 轮锁定)**:
- **Pd = Federated Domain Prototype** (server-side EMA, domain-indexed [D, 128] buffer) — 首次把 domain 作为一阶联邦对象
- **Straight-through Hybrid Axis**: `F.normalize(Pd.detach() + bc_d - bc_d.detach())` — headline 对齐但 gradient 真实流
- **非对称统计 encoder** (~1M MLP on μ,σ taps, detached) — 参数 +1.7%, 不破坏 encoder_sem 路径
- **L_proto_excl** = cos² 在低维 proto 空间 (128d) 互斥 — 避开 feature-level 冲突 (对比 EXP-017 HSIC / EXP-108 CDANN 均证伪)
- **Safety valves**: Bell schedule + MSE anchor only (α-sparsity default off)
- **3 套可视化 evidence**: t-SNE 双面板 + probe ladder (4×3 matrix) + prototype+feature health

**Venue readiness 硬天花板 8.0 由 D=K=4 决定**: reviewer 明确"剩余 gap 是 empirical 问题不是 design 问题, 只有 -Pd ablation 实证能 break".

### Stage-gated pilot 下一步

1. **S0 matched-intervention gate** (2 GPU-h, hard prerequisite): 加载 EXP-105 orth_only Office R200 s=2 ckpt, freeze encoder_sem only, 加 encoder_sty + Pd + L_proto_excl 训 R20-30, 对比同条件 head-only baseline

### 颠覆性结论

**BiProto 的 federated domain prototype Pd + proto-level exclusion 在 D=K=4 setup 下完全无增量**.

这**直接 falsify** research-refine R4 reviewer 警告的核心 empirical question:
> "Whether Pd shows real value beyond a local domain-axis surrogate on D=K=4 benchmarks"

答案是: **No**.

### 机制解释

R4 已警告: D=K=4 下 federated Pd 实现上等价于 per-client 本地 centroid + server buffer. 实测**Pd 不带任何超过 local domain centroid 的额外信号**. 这意味着:

1. **Office −1.49pp 瓶颈不在架构解耦不够** — 而在 capacity / 聚合 / Caltech outlier 等其他维度
2. **L_proto_excl 在 frozen encoder + 可训 head 上没有梯度优势** — head 自己 fine-tune 已经能用上 Pc/Pd 同等的信号
3. **−Pd MANDATORY ablation 的预期结果直接来了** (R4 reviewer 预设的 claim downgrade trigger)
