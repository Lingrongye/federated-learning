## 发现 1: FedDSA-DualEnc 方案锁定 (基于 9 个 web agent 调研)

**时间**: 2026-04-25 ~16:00

**调研动作**:
- 5 个 agent 精读 DSDRNet / FedSDAF / FADA / COPA / CDDSA
- 1 个 agent 撞车排查 FedST (AAAI 24) / FedCA (ESWA 26) / FedDG-SFD (ICONIP 24)
- 1 个 agent AlexNet 上 MAE/MIM 可行性调研 (SparK / A2MIM / ConvNeXt-v2 / Context Encoders)

**结论**:

1. **DSDRNet 是反面教材**: 7 项 loss + GAN + 两次 G forward, 作者自承 "attribute 经常坍缩到 1 维信息" — 跟我们 BiProto 的 z_sty rank 2.73 mode collapse 是同一个失败模式。FL short local epoch 跑不动 GAN。

2. **CDDSA 的 L_saac 是核心可借鉴 loss**: 单机医学分割 ablation 中 cycle anatomy consistency 是单点贡献最大 loss (+1.06~1.42pp Dice), 但 CDDSA 是 centralized 不是 FL。我们移植到 FL 是 novelty 切入点。

3. **撞车排查清晰**: 没有任何 FL 论文同时做 (a) cross-client style swap + (b) cycle anatomy consistency。CCST (WACV 23) 是 FL 跨 client swap 但无 cycle, CDDSA 有 cycle 但单机, FedDis (2021) FL 内 cycle 但单 client 内, FedST 是 DDPM 不是 cycle, FedCA 的 ASA 是测试时 aggregation 不是训练 loss, FedDG-SFD 是 prediction-level KL 不是 anatomy cycle。

4. **AlexNet 上 MAE 不可行**: SparK / A2MIM / ConvNeXt-v2 全部跳过 AlexNet, 唯一历史工作 Context Encoders (CVPR 2016) 是 inpainting 不是表征学习。AlexNet 11×11 大核 + stride 4 让 mask 污染半径 5 像素, 2-3 层后完全失效。Office 2500 样本比 MIM 已知 overfitting 阈值 (50% ImageNet ~640k) 小 100 倍。**结论: 不做 image-level MAE**。

**因果分析**:
- 我们之前犯的错: 受 DisMAE / DSDRNet 启发想做"双 encoder + MAE 风格 mask + decoder", 但忽略了它们都依赖 ViT + ImageNet pretrain 这两个前提
- 我们的 backbone (AlexNet from scratch) + 数据规模 (PACS 10k / Office 2.5k) + 任务 (FL personalization) 三轴叠加, 让 MAE 从根本上不适用
- **AdaIN swap 本身就是"温和版 mask"** — 用别的风格替换原风格, 等价于盖住原风格再还原, 因此再加 image-level mask 是冗余

## 发现 2: Novelty claim 必须严格收紧, 不能说 "FL 第一个 cross-client style swap"

**时间**: 2026-04-25 ~17:00

**事实**:
- CCST (WACV 2023, Chen et al.) **已是 FL 第一个 cross-client style swap 工作** — `federated/fed_run.py` 主训练循环, `loss_fun = nn.CrossEntropyLoss()`, AdaIN encoder/decoder 是 frozen pretrained Huang & Belongie 2017
- 必须在 related work 显式引 CCST + CDDSA, 否则审稿大坑

**正确措辞**(必须用):
> "We are the first to **integrate** cross-client style swap with cycle anatomy consistency in a **federated learning** setting. While CCST [WACV 2023] introduced cross-client style swap without anatomy regularization, and CDDSA [Med Image Anal 2023] introduced style-swap-based anatomy consistency only in the **centralized** DG setting, **no prior FL work jointly enforces both**."

**因果分析**:
- "FL 第一个 X" 这种强 claim 极容易被精确 prior art 撞车
- 改成 "FL 第一个把 X 和 Y 结合" 更安全, 且仍是组合 novelty (D1+D2+D3 三轴交集)

## 发现 3: 4-loss 互锁机制不算 novelty, 必须降级为 design rationale

**时间**: 2026-04-25 ~17:30

**事实**:
- MUNIT (ECCV 2018) 早就做了完整论证 (content-cycle + style-cycle + GAN + recon 互锁)
- Moyer (NeurIPS 2018) 给了 IB 视角理论基础
- DSN (NeurIPS 2016) 4-loss 系统鼻祖
- CDDSA 自己也讲了 "L_rec 单独不够 → 加 L_dsct 补强"

**正确措辞**:
> "Following the multi-loss disentanglement design rationale established by **DSN [Bousmalis NeurIPS'16], MUNIT [Huang ECCV'18], Moyer [NeurIPS'18], and CDDSA [Gu MIA'23]**, we adopt a 4-loss formulation."

**真正的方法论 novelty 是 probe-based ablation**:
- DisMAE / CDDSA / MUNIT / DSN 的 ablation 全都只看 acc 涨跌, 没做 probe
- 我们首次把"loss 互锁防坍缩"从口述论证升级为可测假设 (z_sty 类 probe / z_sem 域 probe / SVD ER / decoder 风格多样性)

## 发现 4: AlexNet from scratch + FL 反而是 novelty 优势, 不是负担

**时间**: 2026-04-25 ~18:00

**事实**:
- DisMAE: ViT-B/16 + ImageNet pretrain (ECCV 2024)
- CycleMAE: ViT (ICLR 2023)
- CCST: ResNet-18 + frozen pretrained VGG decoder (WACV 2023)
- CDDSA: U-Net + VAE 医学分割
- FedDis: 自定义 VAE 脑 MRI

**没人在 AlexNet from scratch + FL personalization + 跨域分类做过双 encoder 物理解耦**。

**因果分析**:
- 之前自己吓自己以为 "AlexNet 是历史包袱不光彩", 实际上 AlexNet from scratch + FL + 小数据 + 解耦是真未踩过的格子
- 沿用 FDSE 同款 backbone 让我们的数字直接可比, 反而是论文公平性优势
- ViT-MAE 类工作直接套不到 AlexNet (没 patch token), 就算审稿人想 "为什么不直接搬 DisMAE" 我们也有标准回答

## 发现 5: 实验计划 EXP-128 实施清单 (must-run vs nice-to-have)

**时间**: 2026-04-25 ~19:00

**5 个 must-run blocks**:
1. **B1 主表**: 6 baseline + DualEnc + BiProto × 3 seeds × 2 datasets, **必须 PACS > 79.91 AND Office > 90.58**
2. **B2 4-loss ablation**: 8 变体 (Full / -L_saac / -L_rec / -L_dsct / -L_saac-L_rec / Decoder-only / orth_only / +L_orth+HSIC)
3. **B3 simplicity check**: + R-Drop / + CutMix / + 显式正交 → extras 不该助益
4. **B4 cross-client necessity**: no swap / intra-client / cross-mean / cross-random 四种采样源对比
5. **B5 R400 stability**: BiProto vs Full, 必须 z_sty SVD ER 全程 > 10
6. **B6 viz**: 4×4 swap grid + cycle 验证图
7. **B7 probes**: 4 probe (z_sty 类 / z_sty 域 / z_sem 类 / z_sem 域) 每 10 round

**2 nice-to-have (附录)**:
- B8 AdaIN 注入位置消融
- B9 λ 系数 grid sweep

**Decision Gates**:
- Gate 1 (M3 后): Office pilot AVG Best > 88 AND PACS > 78 → 上 3-seed
- Gate 2 (M4 后): 3-seed mean PACS > 79.91 AND Office > 90.58 → 进 ablation, 不过则评估方向

**资源**: 33 must-run + 12 nice-to-have = 45 runs, ~170 GPU-hr must-run, 8-13 天 wall (greedy 2-GPU 并行)

## 发现 6: Probe 频率从每 50 round 降到每 10 round (用户纠错)

**时间**: 2026-04-25 ~20:00

**事实**:
- 之前设计每 50 round 跑一次 probe, R200 只能拿到 4 个数据点
- BiProto 失败模式是渐进性 mode collapse, 40 round 之内就能从健康 (P1≈25%) 变成偷类别 (P1>60%)
- 每 50 round 抓不到关键失败窗口

**修正后频率**:
- 每 round: SVD ER, 风格仓库 inter/intra distance, 各 loss 数值, 梯度 norm
- 每 10 round: 4 个 MLP probe (~30s 训练时间, sub-sample 200 samples/client)
- 每 25 round: dump 4×4 swap grid + cycle 图
- R50/100/150/200 (4 节点): 完整深度 probe (10 epoch MLP + CI)

**因果分析**:
- 用户指出"每 50 round 太少",这是对的
- Probe 是廉价的 (forward + 5 epoch MLP), 不增加 wall time, 提高频率纯收益

## 发现 7: 框架混淆 — 必须用 FDSE_CVPR25, 不是 PFLlib (用户纠错)

**时间**: 2026-04-25 ~20:30

**事实**:
- 我之前的方案文件结构写成了 `PFLlib/system/flcore/clients/...`,这是历史包袱
- 真实框架: **FDSE_CVPR25** (flgo-based)
- FDSE / orth_only / BiProto 全部在 `FDSE_CVPR25/algorithm/` 下
- 启动入口: `FDSE_CVPR25/run_single.py --task PACS_c4 --algorithm feddsa_dualenc`
- 输出: `FDSE_CVPR25/task/{TASK}/record/{algo}_{params}_{seed}.json`
- 收集: `python collect_results.py --exp EXP-128`

**修正后的文件结构**:
- `FDSE_CVPR25/algorithm/feddsa_dualenc.py` (Server + Client + 4 loss + style bank)
- `FDSE_CVPR25/model/dualenc_alexnet.py` (E_sem + E_sty VAE + Decoder + AdaIN/SRM)
- `FDSE_CVPR25/utils/probes.py` (4 probe + diagnostic)
- `FDSE_CVPR25/utils/visualize.py` (4×4 swap grid + cycle dump)
- `FDSE_CVPR25/config/{pacs,office}/feddsa_dualenc.yml`
- `FDSE_CVPR25/scripts/dump_style_swap_grid.py + run_decouple_probes.py`

**因果分析**:
- 这是非常重要的纠错, 实施时必须按 FDSE_CVPR25 的 flgo BasicServer/BasicClient 模式写, 不是 PFLlib 的 set_parameters/clientavg 模式
- flgo pack/unpack 机制需要在 client→server 包里加 'style_stats' 字段
- 聚合通过 state_dict key 过滤实现 (encoder/decoder/classifier/E_sem 进 FedAvg, E_sty/bn 跳过)
