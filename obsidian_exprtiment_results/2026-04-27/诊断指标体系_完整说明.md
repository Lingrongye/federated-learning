# PG-DFC / F2DC 诊断指标体系完整说明

## 一句话总结

**训练时录像头 (Hot Path)** 只录原始 binary, 不算任何衍生指标; **后期分析师 (Cold Path)** 看录像算所有衍生指标 (cos sim / t-SNE / silhouette / CKA 等). 同一份录像可以反复 mine 出新指标, **不用重跑训练**.

## 1. 大白话: 我们存了什么 + 能算什么

### 训练时存的"原料" (Hot Path, ~30 MB / 实验)

#### 类型 A: 每 round 元数据 (`round_NNN.npz`, ~50 KB / round)
就像每天记日记, 每 round 存 10 项关键状态:

| 存的东西 | 大白话解释 |
|---|---|
| **sample_shares** | 每个 client 数据量占总量的比例 (FedAvg 默认权重) |
| **daa_freqs** | DaA 计算的实际聚合权重 (跟 sample_shares 比看 DaA 怎么"调整公平") |
| **online_clients** | 这 round 哪些 client 参与训练 (我们都是 全 10 个) |
| **domain_per_client** | 每个 client 的 domain 标签 (photo/art/cartoon/sketch) |
| **per_domain_acc** | 这 round 每个 domain 的全局测试准确率 |
| **global_proto** | server 端汇总的"班级标准答案" prototype (10 类 × 512 维) |
| **local_protos** | 每个 client 自己学到的"个人答案" (10 client × 10 类 × 512 维) |
| **layer_l2** | client 的 conv 跟 global 的差距 (按 layer 分: 浅层 vs 深层) |
| **grad_l2** | client 总 update 强度 = client 训练贡献多大 |
| **all_dataset_names** | 数据集 4 个 domain 名字 |

#### 类型 B: best/final round 重 dump (`best_R*.npz` / `final_R100.npz`, ~14 MB each)
就像在关键时刻 (best round + 最后) 拍证件照, 记录全部细节:

| 存的东西 | 大白话解释 |
|---|---|
| **features** | 全 test 样本 ResNet 提的 512 维 embedding (按 domain 分) |
| **labels** | 全 test 样本的真实 label |
| **preds** | model 预测的 label |
| **logits** | model 预测的 7/10 类 logits (softmax 之前) |
| **confusion** | 4 domain × 7×7 混淆矩阵 (哪些 class 错给哪些 class) |
| **state_dict** | 当时 model 的全部 weights (可以 cold path 重新 evaluate) |
| **domain_names** | 标签 |

### 后期能算的"指标" (Cold Path, 想算就算)

#### 维度 1: Server 聚合行为
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **DaA dispatch ratio** | 每 client 的 daa_freqs / sample_shares (DaA 给放大了多少) | sample_shares + daa_freqs |
| **effective contribution** | α_i × ‖w_i - w_g‖ = "这 client 真实声音多大" | daa_freqs + grad_l2 |
| **per-layer drift** | 浅层 vs 深层 client 偏离多少 | layer_l2 |
| **domain-bucketed dispatch** | 大 domain 跟小 domain 平均权重对比 | sample_shares + domain_per_client |

#### 维度 2: Prototype 行为 (验证 PG-DFC 的核心假设)
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **client-global proto cos sim** | 每 client 的"个人答案"跟"班级共识"有多像 | local_protos + global_proto |
| **proto trajectory over rounds** | proto 从 R1 到 R100 怎么演化 (是 converge 还是 collapse) | round_*.npz 每 round |
| **proto pairwise similarity** | 10 类 proto 之间是否互相像 (mode collapse 检测) | global_proto |
| **per-client per-class proto matrix** (heatmap) | 10 client × 10 class 谁跟 consensus 最像/最远 | local_protos |
| **proto round-to-round change** | proto 跨 round 变化幅度 (早期变化大正常, 后期应稳) | global_proto trajectory |

#### 维度 3: Feature space (representation 质量)
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **t-SNE 2D 可视化** | 把 512 维 feature 降到 2D 看 cluster 形态 | features |
| **silhouette score** | 类内紧密 / 类间分离 score (高 = 好) | features + labels |
| **domain-invariance score** | 同 class 在不同 domain 的 feature 距离 (小 = domain-invariant) | features + labels + domains |
| **CKA cross-method similarity** | 比较两个 method (e.g. PG-DFC vs vanilla F2DC) 学到的 feature 多像 | features (跨实验) |
| **feature norm distribution** | 不同 domain 的 feature L2 范数分布 (异常 = scale 问题) | features |

#### 维度 4: 错例分析
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **per-class accuracy** | 每个 class 单独 accuracy (哪些 class 拖后腿) | confusion matrix |
| **per-domain per-class acc matrix** | domain × class 矩阵 (paper Fig 6 风格) | confusion |
| **confusion pattern** | 哪些 class pair 最容易混 (e.g. dog vs horse) | confusion matrix |
| **logit margin** | 预测对的样本 margin 多大 (model 是否过自信) | logits + labels |

#### 维度 5: 训练稳定性 / 收敛
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **convergence curve** per algo | accuracy 跨 round trajectory | per_domain_acc |
| **best vs last gap** | 训练后期 drift (best > last 多少) | per_domain_acc |
| **per-domain trajectory** | 每个 domain 单独 trajectory (哪些 domain 后期掉) | per_domain_acc |
| **acc cross-round std** | 后 30 round acc std (训练稳不稳) | per_domain_acc |
| **best round per algo** | 最佳 round 是 R几 (paper 数据) | per_domain_acc |

#### 维度 6: Best vs Final 对比 (我们独有, 用 heavy dump 双时刻)
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **t-SNE best vs final 同 method** | 训练后期 feature space 退化多少 (PG-DFC 是不是后期把 webcam 同化了) | best_*.npz + final_R100.npz |
| **silhouette best vs final** | 后期 feature 质量降多少 | features (best 跟 final) |
| **per-class acc shift best→final** | 哪些 class 后期掉 (确认 best 跟 last 都看) | confusion |
| **CKA best-vs-final 同 method** | 后期 representation 漂移多少 | state_dict + features |

#### 维度 7: 跨方法对比 (paper 主图)
| 指标 | 大白话 | 用什么数据 |
|---|---|---|
| **cross-method t-SNE** (4 method × 4 domain) | 不同方法学到的 feature 形态对比 (哪个 method 更 domain-invariant) | features (4 个实验) |
| **cross-method confusion matrix difference** | F2DC vs PG-DFC vs PG-DFC+DaA 的错例对比 | confusion (跨实验) |
| **cross-method per-class acc bar** | 7 class × 4 method matrix | confusion |
| **convergence comparison** | 4 method 收敛速度 + final acc | per_domain_acc trajectories |

## 2. 详细 (学术版): 数据 schema + 计算公式

### `round_NNN.npz` schema (light, ~50 KB)

```python
{
  'round': int,                                  # round index (1-indexed)
  'sample_shares': np.ndarray (K,) float32,      # n_i / sum(n_j), K=parti_num
  'daa_freqs': np.ndarray (K,) float32,          # α_i from Eq (11), 0 if !use_daa
  'online_clients': np.ndarray (K,) int32,       # client indices
  'domain_per_client': np.ndarray (K,) <U16,     # 'photo' / 'art' / ... per client
  'per_domain_acc': np.ndarray (D,) float32,     # global eval per domain
  'all_dataset_names': np.ndarray (D,) <U16,
  'global_proto': np.ndarray (C, dim) float16,   # cross-client class proto, dim=512
  'local_protos': np.ndarray (K, C, dim) float16,# per client per class proto
  'layer_l2_pickle': np.ndarray (1,) object,     # JSON {client_id: {layer: l2_norm}}
  'grad_l2': np.ndarray (K,) float32,            # ‖w_i - w_g‖_2 per client
}
```

### `best_R{NNN}.npz` / `final_R100.npz` schema (heavy, ~14 MB)

```python
{
  'round': int,
  'current_acc': float,
  'features': dict[domain_name -> np.ndarray (N_d, 512) float16],
  'labels':   dict[domain_name -> np.ndarray (N_d,) int32],
  'preds':    dict[domain_name -> np.ndarray (N_d,) int32],
  'logits':   dict[domain_name -> np.ndarray (N_d, C) float16],
  'confusion': dict[domain_name -> np.ndarray (C, C) int32],
  'state_dict': dict[layer_key -> np.ndarray fp16],
  'domain_names': np.ndarray (D,) <U16,
}
```

### 关键计算公式

#### DaA dispatch ratio
```python
ratio_i = daa_freqs[i] / sample_shares[i]
# > 1: client 被升权 (DaA 觉得它"应该多发声")
# < 1: client 被降权 (DaA 觉得它"应该少发声")
# = 1: 跟 FedAvg 完全相同
```

#### client-global proto cos similarity (验证假设 A: PG-DFC 是否同化 client)
```python
for each client i, class c:
    cos_ic = dot(local_proto[i,c], global_proto[c]) / (norm(local_proto[i,c]) * norm(global_proto[c]))
# 平均: cos_per_client[i] = mean over class c of cos_ic
# 解读:
#   cos > 0.85 = client 被高度同化, 几乎没自己的"个性"
#   cos in [0.5, 0.85] = 健康的"既学共识又保留个性"
#   cos < 0.5 = client 太"叛逆" (可能 outlier domain)
```

#### effective contribution (验证假设 C: DaA 是否真改变 effective dispatch)
```python
contrib_i = daa_freqs[i] * grad_l2[i]
# 跟 vanilla FedAvg 的 sample_shares[i] * grad_l2[i] 比, 看 DaA 真实改变了 dispatch 还是只改了表面
```

#### proto collapse score (mode collapse 检测)
```python
# 算 10 类 proto 之间的 pairwise cos sim
sim_matrix = cosine_similarity(global_proto)  # (C, C)
collapse = (sim_matrix.sum() - C) / (C * (C-1))  # off-diagonal mean
# 越接近 1: 类 proto 越相似 → mode collapse
# 越接近 0: 类 proto 互相正交 (健康)
```

#### t-SNE on heavy snapshot
```python
import sklearn.manifold
all_feats = np.concatenate([data['features'][d].astype(np.float32) for d in domains])
all_domains = np.concatenate([[d]*len(data['features'][d]) for d in domains])
all_classes = np.concatenate([data['labels'][d] for d in domains])
embedding = TSNE(n_components=2, perplexity=30).fit_transform(all_feats)
# plot: 4 colors (domain) × 7 markers (class)
```

#### silhouette by class / domain
```python
# 类内紧密度 vs 类间分离度
sil_class = silhouette_score(features, labels)         # 高 = class 区分好
sil_domain = silhouette_score(features, domain_ids)    # 高 = domain 区分明显, 低 = domain-invariant
```

#### CKA cross-method similarity
```python
def cka(X, Y):
    # X, Y: (N, D) feature matrix
    Kx = X @ X.T
    Ky = Y @ Y.T
    Kx -= Kx.mean(0); Ky -= Ky.mean(0)
    return (Kx * Ky).sum() / (np.sqrt((Kx*Kx).sum()) * np.sqrt((Ky*Ky).sum()))
# > 0.8: 两个 method 学到非常相似的 representation
# < 0.5: 两个 method 学到不同的 representation
```

## 3. 我们用这些指标回答的关键问题

### Q1: PG-DFC 真的把 webcam 同化了吗?
- **看**: `client_global_proto_cos[webcam_client_id]` trajectory over rounds
- **结论**: 如果 webcam client 的 cos sim 一路涨到 > 0.85 → 同化假设成立

### Q2: DaA 跟 PG-DFC 是不是机制重叠?
- **看 1**: vanilla F2DC vs F2DC+DaA 的 `effective_contribution[dslr_client]` — DaA 是不是真升权 dslr?
- **看 2**: vanilla PG-DFC 的 `client_global_proto_cos[dslr_client]` — PG-DFC 是不是 implicit 拉 dslr 接近 consensus?
- **结论**: 如果 PG-DFC 的 dslr 已经接近 consensus, DaA 升权 dslr 没增量 → 重叠

### Q3: F2DC+DaA 的 +3pp 来自哪?
- **看**: per-class confusion delta (vanilla vs DaA)
- **找**: 哪些 class 在哪些 domain 涨最多

### Q4: PG-DFC 的 best-vs-last drop 5pp 是什么 drift?
- **看**: best_R{X}.npz vs final_R100.npz 的 t-SNE
- **看**: confusion matrix 哪些 class 后期掉

### Q5: 是不是 webcam client 的 update 被 wash 掉?
- **看**: `effective_contribution[webcam_client]` over rounds — 如果一直小, 说明 webcam 声音被 wash
- **看**: DaA 时 webcam 是不是被升权 (daa_freqs[webcam])
