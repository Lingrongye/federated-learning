# Research Proposal Round 1: Self-Masked Style-Weighted Multi-Positive InfoNCE (SCPR)

## Problem Anchor (preserved verbatim)

- Bottom-line: Cross-domain FL where clients hold different visual styles; existing prototype alignment averages per-class prototypes globally and broadcasts, diluting outlier-domain clients (Caltech, Sketch).
- Must-solve:
  1. Global-mean prototype washes out style → outlier clients lose their structure.
  2. SAS (param-space routing) succeeds on Office single-outlier (+1.21%) but fails on PACS full-outlier (-0.65%) because param-space softmix degenerates to FedBN-local (EXP-086).
  3. FedDSA "Share" block never worked: EXP-059 (z_sem AdaIN) broke PACS -2.54%; EXP-078d (h-space AdaIN -> InfoNCE) NaN.
- Non-goals: no style-as-augmentation, no classifier personalization, no auxiliary losses, no architecture changes, no new trainable components.
- Constraints: ResNet-18 / AlexNet, FedBN, R=200, 3 seeds, orthogonal decouple kept, InfoNCE target is the ONLY thing that changes.
- Success: PACS AVG Best 3-seed mean >= 81.5% (beat M3 isolated 81.91%), Office AVG Best >= 90.5% (beat SAS 89.82%), drop <= 2%, <100 LOC, 0 new trainable.

## Technical Gap

No existing method uses "client style as retrieval key" over "domain-indexed class prototypes as values" in the decoupled representation space. Prior alternatives either erase style (FDSE/FedSeProto/FedDP), keep style local (FedSTAR/FedBN), do AdaIN in mixed space (FISC/PARDON, disproved for us in EXP-059), or route in parameter space (SAS, fails on PACS full-outlier).

## Method Thesis

Replace M3's equal-weighted multi-positive InfoNCE positives with style-conditioned weights, and self-mask (j != i) so Share never collapses to self-only. This is the smallest adequate change to close FedDSA's Share gap.

## Contribution Focus

- SINGLE main contribution: Self-Masked Style-Weighted Multi-Positive InfoNCE (SCPR).
- Appendix composability check: SCPR + SAS on Office (NOT a main claim).
- Explicit non-contributions: no style-as-training-data, no param routing, no new loss terms.

## Proposed Method

### Complexity Budget
- Backbone / sem_head / style_head / classifier / BN: reused (frozen).
- Orthogonal decouple (cos^2 + HSIC): reused (Plan A).
- Style bank (z_sty per-client proto): reused from SAS/EXP-084.
- Class-proto bank (per-(class, client)): reused from M3/EXP-072.
- New: SCPR attention function (~20 LOC) + self-mask & per-class renormalize (~10 LOC).
- New trainable components: 0. New loss terms: 0. New hyperparameters: 1 (tau_SCPR, default 0.3 inherited from SAS).

### Core Mechanism — Self-Masked Style-Weighted Multi-Positive InfoNCE

Setup (matches M3 up to weight choice):
- Sample (x_i, y_i=c) on client k, z_i = sem_head(backbone(x_i))
- Server provides {p_c^j}_{c=1..C, j=1..K} and {style_proto_j}_{j=1..K}

Self-masked style attention:
  g_kj = cos(style_proto_k, style_proto_j),  j = 1..K
  raw_j = exp(g_kj / tau_SCPR) * 1[j != k]            # self-mask

Per-class renormalize (handle missing class):
  A_k^c = { j : j != k, p_c^j exists }
  w_{k->j}^c = raw_j / sum_{l in A_k^c} raw_l    if j in A_k^c
  w_{k->j}^c = 0                                  otherwise

SCPR loss (for sample z_i with y_i = c, on client k):
  L_SCPR(z_i)
    = - sum_{j in A_k^c} w_{k->j}^c * log [
          exp(sim(z_i, p_c^j) / tau_nce)
          / ( exp(sim(z_i, p_c^j) / tau_nce)
              + sum_{c' != c, l in A_k^{c'}} exp(sim(z_i, p_{c'}^l) / tau_nce) )
        ]

All p_c^j and style_proto_j are detached (soft anchors).

### Mathematical properties (kept to ONE)

- tau_SCPR -> infinity: w_{k->j}^c = 1/|A_k^c| -> strictly reduces to M3 domain-aware multi-positive InfoNCE (verified +5.09% on PACS).
- tau_SCPR -> 0: one-hot to argmax_{j != k} cos(s_k, s_j), i.e., nearest-style single positive.
- Self-mask: Share path never uses self prototype, so w_kk is always 0.

### Warmup handling

- Round < warmup (50): bank not yet stable; fallback to uniform w (== M3 equal multi-pos == tau -> infinity branch).
- Round >= warmup: full SCPR.
- Bank update: clients upload style_proto & class_proto in pack(); server overwrites bank in iterate().

### Failure modes & fallbacks

| Failure | Signal | Fallback |
|---------|--------|----------|
| Attention uniform collapse | H(w_k) > 0.95 * log(K-1) for >20 rounds | L2-normalize style key and re-project (still no new param) |
| NaN / numerical instability | loss NaN | tau_SCPR 0.3 -> 1.0 |
| PACS < M3 | 3-seed AVG Best < 81.91% | check self-mask & attention entropy |
| A_k^c empty | no other client has class c | set L_SCPR = 0 for that sample (CE-only fallback) |

Built-in safety: uniform-w case is strictly M3 (+5.09%). Worst-case recovery bound holds mathematically.

### Novelty & elegance

- Single mechanism (self-masked style-weighted multi-positive InfoNCE), 0 new params, <100 LOC.
- Math: uniform w = M3, tau -> 0 = nearest-style share; no ambiguous generalization claims.
- First method to sit in the (Decouple, Style-share) cell of the 2x2 matrix.
- Closes the FedDSA Share narrative with mechanism-level contribution.
