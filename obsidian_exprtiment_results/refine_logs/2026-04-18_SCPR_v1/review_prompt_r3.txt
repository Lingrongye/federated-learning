# Research Proposal Round 2: Self-Masked Style-Weighted Multi-Positive InfoNCE (SCPR)

## Problem Anchor (preserved verbatim)

- Cross-domain FL where clients hold different visual styles; existing prototype alignment averages per-class prototypes globally and broadcasts, diluting outlier-domain clients.
- Must-solve: (1) global-mean washes out style, (2) SAS param-space routing fails on PACS full-outlier (EXP-086 -0.65%), (3) FedDSA Share block never landed (EXP-059 -2.54%, EXP-078d NaN).
- Non-goals: no style-as-data, no classifier personalization, no aux losses, no architecture change, no new trainable components.
- Constraints: ResNet-18/AlexNet, FedBN, R=200, 3 seeds, orth decouple kept.
- Success: PACS AVG Best 3-seed >= 81.5% (beat M3 81.91%); Office >= 90.5% (beat SAS 89.82%); drop <= 2%; <100 LOC; 0 new trainable.

## Technical Gap

No existing method uses *client style as retrieval key* over *domain-indexed class prototypes as values* in the decoupled representation space. Alternatives either erase, keep-local, or do AdaIN in mixed space.

## Method Thesis

Replace M3's equal-weighted multi-positive InfoNCE positives with style-conditioned weights, self-masked (j != i) by construction. Smallest adequate change to close the Share gap.

## Contribution Focus

- SINGLE main contribution: Self-Masked Style-Weighted Multi-Positive InfoNCE (SCPR).
- Appendix composability check (not main): SCPR+SAS on Office.
- Non-contributions: style-as-data, param routing, new loss terms.

## Proposed Method

### Notation (unified)

- s_k := normalize(E_{x in D_k}[z_sty(x)])  -- L2-normalized client style anchor
- p_c^j := E_{(x, y=c) in D_j}[z_sem(x)]     -- per-(class, client) prototype
- tau_nce = 0.3 (Plan A optimal)
- tau_SCPR = 0.3 (default, inherited from SAS)

### Complexity Budget

- Backbone / sem_head / style_head / classifier / BN: reused, frozen.
- Orth decouple (cos^2 + HSIC): reused (Plan A).
- Style bank {s_j}: reused from SAS.
- Class-proto bank {p_c^j}: reused from M3.
- NEW: SCPR attention + self-mask + per-class renormalize, ~30 LOC.
- 0 new trainable components, 0 new loss terms, 1 new hyperparameter (tau_SCPR).

### Core Mechanism (single canonical formula)

Self-masked style attention:
  g_kj = cos(s_k, s_j),   j = 1..K
  raw_j = exp(g_kj / tau_SCPR) * 1[j != k]

Per-class renormalize:
  A_k^c = { j : j != k, p_c^j exists }
  w_{k->j}^c = raw_j / sum_{l in A_k^c} raw_l    if j in A_k^c
  w_{k->j}^c = 0                                  otherwise

SCPR loss for (z = sem_head(backbone(x)), y = c) on client k:
  L_SCPR(z, c)
    = - sum_{j in A_k^c} w_{k->j}^c * log [
          exp(sim(z, p_c^j) / tau_nce)
          / ( exp(sim(z, p_c^j) / tau_nce)
              + sum_{c' != c, l in A_k^{c'}} exp(sim(z, p_{c'}^l) / tau_nce) )
        ]

All p_c^j and s_j are detached.

Total loss: L = L_CE + lambda_orth * L_orth + lambda_hsic * L_HSIC + lambda_sem * L_SCPR
(weights 1.0, 0.1, 1.0 inherited from Plan A)

### Mathematical Property (single claim)

tau_SCPR -> infinity: w_{k->j}^c = 1/|A_k^c| -> SCPR strictly reduces to M3 domain-aware multi-positive InfoNCE (verified +5.09% on PACS).

Other limits:
- tau_SCPR -> 0: one-hot to argmax_{j != k} cos(s_k, s_j), nearest-style single positive.
- self-mask: w_{k->k} == 0 by construction; Share path cannot collapse to FedBN.

### Why style weighting beats uniform: a signal-to-noise perspective

**Setup**: After FedDSA orthogonal decouple, z_sem theoretically keeps only semantic info and z_sty only style. But cos^2(z_sem, z_sty) -> 0 is a *soft* constraint; in practice we measure residual cos(z_sem, z_sty) ~ 0.05-0.1 at training end.

**Implication**: The class prototype p_c^j = E[z_sem | y=c, D_j] absorbs some of client j's style residue under imperfect decouple. Larger style_dist(k, j) implies higher *style noise component* of p_c^j from client k's perspective.

**M3 uniform multi-positive flaw**: In SupCon, each positive contributes equal pulling force, implicitly assuming all positives have equal class-signal-to-style-noise ratio. But in reality, far-style p_c^j has lower SNR. On PACS:
- Pulling Art's z_sem toward Cartoon p_dog (near style) -> mostly "art-simplified dog" semantic signal
- Pulling Art's z_sem toward Sketch p_dog (far style) -> absorbs "extreme line-art" style residue

Equal-weighting means Art's z_sem is contaminated by Sketch style noise.

**SCPR's correction (derived from decouple imperfection)**:
w_{k->j}^c ∝ exp(cos(s_k, s_j) / tau)

Near-style prototypes receive high weight (clean signal); far-style receive low weight (noisy signal). This is **importance weighting by SNR over positives**, not an arbitrary reweighting:
- When decouple is perfect: style_dist does not affect p_c^j quality -> uniform weight optimal (SCPR -> M3).
- When decouple is imperfect (always in practice): style weighting is a **necessary correction** for residual coupling.

**Conceptual position**: SCPR sits between M3 (style-ignorant) and SAS (param personalization). It weights alignment targets by style WITHOUT forcing parameter personalization. This explains:
- Why SAS fails on PACS full-outlier: all clients' params degenerate locally.
- Why SCPR works on both PACS (param shared -> no local degeneracy) and Office (style-aware alignment -> single-outlier gains).
