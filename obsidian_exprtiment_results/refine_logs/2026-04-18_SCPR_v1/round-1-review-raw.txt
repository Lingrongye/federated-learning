## Proposed Method

### Complexity Budget

| 组件 | 状态 |
|------|------|
| 主干 (ResNet-18/AlexNet) | 冻结 |
| 正交解耦 (cos² + HSIC) | 复用 (Plan A) |
| sem_head / style_head / classifier | 复用 (完全不个性化) |
| BN | 复用 (FedBN) |
| Style bank / Class-proto bank | 复用 |
| SCPR retrieval 模块 | 新增 (仅 attention 函数,无参数) |
| Style-weighted multi-pos InfoNCE | 新增 (仅 loss 权重调整,无参数) |

新 trainable 组件 0 个, 新 loss 项 0 个。

### System Overview

Client k: x -> backbone -> h(1024d) -> [style_head -> z_sty -> style_proto_k; sem_head -> z_sem -> classifier -> y_hat]
                                                                       |
                                                                       v InfoNCE target
                                                        L_InfoNCE(z_sem, P*)

Server:
  Style-proto bank {s_k}_{k=1..K}; Class-proto bank {p_c^k}_{c,k}
  For each target client i:
    w_{i->j} = softmax_j(cos(s_i, s_j) / tau_SCPR)
    P*_c^i  = Sum_j w_{i->j} * p_c^j
  Dispatch {P*_c^i} to client i (replaces global mean)
  DFE / sem_head / classifier: standard FedAvg
  BN: not aggregated (FedBN)

### Core Mechanism - SCPR Retrieval

Input:
- Client k style s_k = (mu_k, sigma_k) in R^{2d}, d=512 for ResNet-18 (1024 total)
- Server maintains {s_k, {p_c^k}_c}_{k=1..K}

Retrieval:
  raw cos:            g_ij = cos(s_i, s_j) in [-1, 1]
  tempered softmax:   w_{i->j} = exp(g_ij / tau_SCPR) / sum_l exp(g_il / tau_SCPR)
  retrieved proto:    P*_c^i = sum_j w_{i->j} * p_c^j

Properties:
- tau_SCPR -> infinity: uniform w, degenerates to FedProto global mean
- tau_SCPR -> 0: one-hot w, degenerates to nearest-style retrieval
- Default tau_SCPR = 0.3 (inherit SAS optimal); ablation {0.1, 0.3, 1.0, 3.0}

Gradient detach: P*_c^i.detach() - retrieved proto is a soft anchor, no gradient back to proto bank.
This is the key safety design avoiding EXP-078d NaN (which leaked gradient through AdaIN-modified z_sem).

### Supporting Component - Style-Weighted Multi-Positive InfoNCE (optional)

Instead of collapsing K prototypes into one weighted mean, let all K prototypes be positives with weighted pulling:

L_SCPR(z_i, y=c) = - Sum_j w_{i->j} * log [ exp(sim(z_i, p_c^j)/tau_nce) / (exp(sim(z_i, p_c^j)/tau_nce) + Sum_{c'!=c, l} exp(sim(z_i, p_{c'}^l)/tau_nce)) ]

- Uniform w: reduces to SupCon multi-positive (= M3)
- One-hot w: reduces to single-positive nearest-style
- Middle: style-weighted pulling

Default main implementation is single retrieved proto (simpler); weighted multi-pos is an ablation.

### Modern Primitive Usage

Intentionally conservative: no LLM/VLM/Diffusion/RL.
Reasons: (1) the problem is pure FL communication + prototype alignment, (2) CLIP/VLM violates fair baseline comparison, (3) CVPR FL community values mechanism cleanliness. Frontier leverage is expressed through attention-based retrieval (Transformer-era primitive) applied to FL prototype systems.

### Integration

- Hook point: replace `FDSE_CVPR25/algorithm/feddsa_scheduled.py` iterate() prototype aggregation branch
- Frozen: backbone / sem_head / style_head / classifier / BN, FedAvg/FedBN unchanged
- New: Server `_retrieve_prototypes(target_cid)` (~30 lines); Server removes proto_global mean; Client InfoNCE target reads from retrieved_protos (~5 lines)
- Client training flow unchanged: z_sem = sem_head(h); L_CE = CE(classifier(z_sem), y); L_InfoNCE = infonce(z_sem, retrieved_protos)

### Training Plan

Budget: R=200, E=1(Office)/E=5(PACS), B=50, LR=0.05.

Hyperparameters (minimal sufficient):
- lambda_orth = 1.0, lambda_hsic = 0.1, lambda_sem = 1.0 (Plan A)
- tau_nce = 0.3, tau_scpr = 0.3
- warmup = 50

Monitoring (diagnostic, not training):
- H(w_k) = -sum_j w_{k->j} log w_{k->j} (collapse detection)
- L2(retrieved_proto, global_mean) (degeneracy detection)
- cos_sim(grad_CE, grad_InfoNCE) (gradient conflict)

### Failure Modes and Diagnostics

| Failure | Signal | Fallback |
|---------|--------|----------|
| Attention uniform collapse | H(w) > 0.95*log(K) for >20 rounds | project style key to lower dim, or tau -> 0.1 |
| One-hot collapse | H(w) < 0.1*log(K) | tau -> 1.0 |
| PACS regression below baseline | 3-seed AVG Best < orth_only | verify >= M3 uniform attention; else 50/50 mix with global mean |
| InfoNCE NaN | loss NaN/Inf | more aggressive detach, tau_nce 0.3 -> 0.5 |
| Self-attention dominance (w_kk > 0.9) | collapse to FedBN | mask self (j != k) |

Built-in safety: worst case SCPR with uniform attention = M3 (already +5.09%).

### Novelty and Elegance Argument

Closest prior works:
1. FedProto (AAAI 2022): SCPR replaces its target construction via style attention
2. FPL (CVPR 2023) / FedPLVM (NeurIPS 2024): both use prototype-self geometry as cluster key; SCPR uses client style as retrieval key (semantically different)
3. FedSTAR: Transformer aggregator + FiLM, but style strictly LOCAL; SCPR is first to make style a cross-client attention key
4. FISC / PARDON (ICDCS 2025): share style via AdaIN in image/mixed feature space; SCPR shares style in decoupled representation space via target construction
5. SAS (ours, unpublished): param-space routing, fails on PACS full-outlier; SCPR moves routing to prototype layer, keeps all clients on same sem_head

2x2 differentiation matrix:

|   | No style sharing | Style sharing |
|---|-----------------|---------------|
| No decouple | FedBN / FedAvg / FedProto | FISC / PARDON / StyleDDG / FedCCRL |
| Decouple | FedSTAR / FedSeProto / FDSE | **SCPR (first)** |

Elegance:
- Single mechanism, 0 new params, <100 lines
- Math: strictly generalizes FedProto (tau->inf) and M3 (uniform w, multi-pos variant)
- Worst case = M3 (+5.09%)
- Closes FedDSA Share narrative; avoids all known failure modes
