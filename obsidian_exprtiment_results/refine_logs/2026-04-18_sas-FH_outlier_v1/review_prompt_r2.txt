# Research Proposal: Style-Conditioned Classifier Routing for Outlier-Domain Recovery in Federated Domain Generalization

## Problem Anchor

- Bottom-line problem: FedDSA Plan A (sas personalizes only `semantic_head`; classifier head and backbone go through FedAvg) fails on outlier domains. PACS art_painting 65.69%, Office Caltech 77.68% (other domains much higher). 70 art misclass all conf >= 0.85, concentrated on person<->dog<->horse<->guitar.
- Must-solve bottleneck: Shared classifier dilutes outlier-domain decision boundary by majority preference.
- Non-goals: No backbone change, no new trainable modules, no inference-time tricks, no per-sample sas.
- Constraints: 3-seed {2,15,333} x R200 x E=1 aligned with FDSE; SC2 single GPU 2-4h/run; code <= 50 lines.
- Success: art/Caltech +2% AVG Best; other domains drop < 1%; Office overall >= FDSE 90.58.

## Technical Gap

Plan A sas covers only the projection head `semantic_head`. The final FC classifier (1024 -> num_classes) still uses FedAvg and is the only layer deciding the class decision boundary. Outlier clients get personalized feature mapping but no personalized boundary -> feature right, boundary wrong.

Missing mechanism: a signal that decides *which clients should share classification boundaries*. FedROD uses uniform per-client classifier (no conditioning). FedAlign uses classifier personalization + feature alignment (no style signal). **We claim style similarity is the natural routing signal for classifier-boundary sharing.**

Smallest adequate: move `classifier.*` keys into sas_keys -- zero new parameters, ~30 lines.

Core claim: *Style similarity is the natural routing signal for sharing classification decision boundaries; classifier-level personalization is the minimal sufficient granularity under style-driven outlier domains.*

## Method Thesis

- One-sentence thesis: **Style similarity is the right routing signal for deciding which clients should share classification decision boundaries**, realized by extending style-aware aggregation from the semantic projection head alone to the full head chain (projection + classifier), with zero new trainable components.
- Why smallest adequate: server-side pack() change, ~30 lines, no new modules / losses / augmentations.
- Why timely: FedROD (ICLR 23), FedAlign (CVPR Workshop 25), DualFed already established per-client classifier as pFL. Our contribution: **conditioning the classifier personalization on style similarity**, not uniform/random.

## Contribution Focus

- Dominant contribution: Style-conditioned classifier routing (sas-FH).
- Optional supporting contribution: **NONE**. (The former granularity sweep is absorbed into Claim 1 counterfactuals; not a parallel contribution.)
- Explicit non-contributions: No novelty claim on classifier personalization itself (FedROD did), no MixStyle / SupCon / focal loss, no BN strategy change, no per-sample / multi-centroid aggregation.

## Proposed Method

### Complexity Budget
- Frozen / reused: Plan A architecture, style_proto extraction, FedBN BN-local, L_orth, sas softmax, all hyperparameters.
- Changed: server pack() aggregation path for `classifier.weight/bias` -- from FedAvg to sas-personalized.
- New trainable components: **0**.
- Intentionally excluded: per-client sas_tau, style_head personalization, backbone sas, encoder last-block sas, multi-centroid (different paper).

### System Overview

Client i: encoder(x) -> h; h -> sem_head -> z_sem -> head -> logits; h -> style_head -> z_sty -> style_proto. Local SGD with CE + L_orth. Upload encoder/sem_head/style_head/head/style_proto.

Server: maintains client_sem_states[j], client_head_states[j] [NEW], client_style_protos[j]. For target client i: w_ij = softmax(cos(sp_i, sp_j)/tau); per_sem_i = sum_j w_ij * sem_state[j]; per_head_i = sum_j w_ij * head_state[j] [NEW]; encoder/style_head/BN via standard FedAvg. Send personalized (encoder, sem_head_i, head_i) to client i.

### Core Mechanism

- Change: `sas_keys = [k for k in global_dict if k.startswith('semantic_head.') or k.startswith('head.')]`
- Aggregation rule unchanged: w_ij = softmax(cos(sp_i, sp_j)/sas_tau); per_param_i[k] = sum_j w_ij * client_state[j][k] for k in sas_keys.
- Client-side training unchanged.

### Modern Primitive Usage
NONE. Confirmed appropriate by R1 review.

### Integration into Pipeline
File: `FDSE_CVPR25/algorithm/feddsa_scheduled.py` pack() method. Config flag `sas=2` (0=off, 1=sem-only, 2=sem+head).

### Training Plan
Single-stage joint training (same as Plan A). Office-Caltech10 c4 main; PACS c4 negative-control only. CE + lambda_orth * L_orth. R200 E=1 B=50 LR=0.05 decay 0.9998. Seeds {2,15,333}.

### Failure Modes and Diagnostics
- Failure 1: Caltech no gain -> use Claim 2 swap diagnostic to diagnose; if swap also shows no effect -> classifier not bottleneck -> halt.
- Failure 2: training unstable (drop > 3%) -> retry tau=0.5.

### Novelty and Elegance Argument
Closest work: FedROD (uniform per-client classifier), FedAlign (classifier + feature alignment), FedDSA Plan A (sas on sem_head only). Exact difference: **style similarity as the conditioning signal for classifier sharing** -- not uniform, not feature-alignment-coupled, not just-projection-head. Paper story (1 sentence): "Style similarity is the natural conditioning signal for sharing classification decision boundaries under domain heterogeneity."

### Claim 2 (Mechanism Diagnostic, zero-training): feature right, boundary wrong

Load A2 R150 best checkpoint. For each client i:
- Condition (a): personalized encoder + personalized sem_head + personalized head_i
- Condition (b): personalized encoder + personalized sem_head + **global FedAvg** head

Predicted signature:
- Caltech: (a) > (b) by >= 2%
- Other Office domains: (a) ~ (b), delta < 0.5%

If signature holds: direct proof that classifier sharing hurts outlier specifically, localized to the head layer.
