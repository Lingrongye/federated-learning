# Research Proposal: Style-Conditioned Classifier Routing for Outlier-Domain Recovery in Federated Domain Generalization

## Problem Anchor

- Bottom-line problem: FedDSA Plan A fails on outlier domains. PACS art 65.69%, Office Caltech 77.68%. 70 art misclass all conf >= 0.85.
- Must-solve bottleneck: Shared classifier dilutes outlier-domain decision boundary.
- Non-goals: No backbone change; no new trainable modules; no inference tricks; no per-sample sas.
- Constraints: 3-seed {2,15,333} x R200 x E=1 aligned with FDSE; SC2 single GPU 2-4h/run; code <= 50 lines; comparable with EXP-083/084/086.
- Success: Caltech +2% AVG Best; other domains drop < 1%; Office AVG Best >= FDSE 90.58.

## Technical Gap

Plan A sas only covers `semantic_head`; classifier `head` uses FedAvg. Outlier clients get personalized features but all share one decision plane -> feature right, boundary wrong.

Missing mechanism: a signal that decides *which clients should share classification boundaries*. FedROD = uniform per-client classifier. FedAlign = classifier personalization + feature alignment (no style signal). We claim **style similarity is the natural routing signal for classifier-boundary sharing**.

Smallest adequate: move `classifier.*` into sas_keys. Zero new parameters.

## Method Thesis

One-sentence: **Under style heterogeneity, the key federated-learning question at the classifier layer is not whether to personalize but how to route the sharing, and style similarity is the natural routing signal.**

Realization: extend FedDSA Plan A's sas from `semantic_head` to the full head chain (projection + classifier).

## Contribution Focus

Dominant: Style-conditioned classifier routing (sas-FH). Supporting: NONE.

## Proposed Method

### Config switch (sas flag 0..4)

- sas=0: baseline no sas (B0 = EXP-083)
- sas=1: sem_head sas only, classifier FedAvg sample-weighted (B1 = Plan A = EXP-084)
- sas=2: sem_head + classifier, both style-conditioned (OURS, A2)
- sas=3: sem_head sas + classifier uniform-avg (C2)
- sas=4: sem_head sas + classifier fully local (C1)

### Aggregation (for k in sas_keys, identical participation set S_t per round)
- A2: per_param_i[k] = sum_{j in S_t} w_ij * client_state[j][k], w_ij = softmax(cos(sp_i, sp_j)/sas_tau)
- C2: per_param_i[k] = (1/|S_t|) * sum_{j in S_t} client_state[j][k]  -- same head for every i
- C1: per_param_i[k] = client_state[i][k]  -- local only

Office c4 proportion=1.0 so S_t = {0,1,2,3} for all rounds. A2 and C2 aggregate over the exact same participation set; only the weighting differs.

Client-side training unchanged: CE + lambda_orth * L_orth.

### Integration
FDSE_CVPR25/algorithm/feddsa_scheduled.py pack(). sas flag extended to 0..4. ~40 lines incl. branches and unit tests.

### Training
R200 E=1 B=50 LR=0.05 decay=0.9998. Seeds {2,15,333}. Office-Caltech10 c4 main; PACS c4 appendix-only.

### Failure Modes
- Caltech no gain -> Claim 2 swap diagnostic; if swap null -> halt.
- Training instability -> sas_tau=0.5 retry.

### Modern Primitive Usage
NONE.

### Novelty
vs FedROD: style conditioning, not uniform. vs FedAlign: no feature alignment added. vs Plan A: covers full head chain, framed as routing problem.

Paper story: "Classifier aggregation in federated DG is a routing problem, not a scope problem. Style similarity provides the routing."
