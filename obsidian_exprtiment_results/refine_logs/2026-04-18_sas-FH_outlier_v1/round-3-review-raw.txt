# Research Proposal: Style-Conditioned Classifier Routing for Outlier-Domain Recovery in Federated Domain Generalization

## Problem Anchor

- Bottom-line problem: FedDSA Plan A fails on outlier domains. PACS art 65.69%, Office Caltech 77.68%. 70 art misclass all conf >= 0.85, concentrated on person<->dog<->horse<->guitar.
- Must-solve bottleneck: Shared classifier dilutes outlier-domain decision boundary by majority preference.
- Non-goals: No backbone change, no new trainable modules, no inference-time tricks, no per-sample sas.
- Constraints: 3-seed {2,15,333} x R200 x E=1 aligned with FDSE; SC2 single GPU 2-4h/run; code <= 50 lines.
- Success condition: Caltech +2% AVG Best; other domains drop < 1%; Office AVG Best >= FDSE 90.58. PACS only if Office succeeds.

## Technical Gap

Plan A's sas covers only `semantic_head`; classifier `head` still uses FedAvg. Outlier clients get personalized features but all share one decision plane -> feature right, boundary wrong.

Missing mechanism: a signal that decides *which clients should share classification boundaries*. FedROD uses uniform per-client classifier (no conditioning). FedAlign couples classifier personalization with feature alignment (no style signal). **We claim: style similarity is the natural routing signal for classifier-boundary sharing.**

Smallest adequate: move `classifier.*` into sas_keys. Zero new parameters.

## Method Thesis

- **One-sentence thesis**: Under style heterogeneity, the key federated-learning question at the classifier layer is not *whether* to personalize but *how to route* the sharing — and style similarity is the natural routing signal.
- Concrete realization: Extend FedDSA Plan A's sas from `semantic_head` to the full head chain (projection + classifier); zero new trainable components.
- Why timely: FedROD (ICLR 23), FedAlign (CVPR Workshop 25), DualFed established per-client classifier as pFL. Our contribution: **conditioning the classifier personalization on style similarity**.

## Contribution Focus

- Dominant contribution: Style-conditioned classifier routing (sas-FH), framed as a routing problem at the classifier layer.
- Optional supporting contribution: **NONE**.
- Explicit non-contributions: No novelty on classifier personalization itself; no MixStyle/SupCon/focal; no BN strategy change; no per-sample / multi-centroid.

Config switch (sas flag):
- sas=0 : no personalization (baseline B0 = EXP-083)
- sas=1 : sem_head sas only, classifier FedAvg (B1 = Plan A = EXP-084)
- sas=2 : sem_head + classifier, both style-conditioned (OURS, A2)
- sas=3 : sem_head sas + classifier uniform-avg (C2 counterfactual)
- sas=4 : sem_head sas + classifier fully local (C1 counterfactual)

Aggregation rule (for k in sas_keys):

- C1 (sas=4): per_param_i[k] = client_state[i][k]  — local only, no sync

Client-side training unchanged: CE + lambda_orth * L_orth.
