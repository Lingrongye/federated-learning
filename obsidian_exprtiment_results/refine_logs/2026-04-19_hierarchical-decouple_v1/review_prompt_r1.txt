# Research Proposal Round 0: Hierarchical IBN-Augmented Decoupling (HIBD)

## Problem Anchor

- **Bottom-line**: Under K=4 FedBN cross-domain FL, current FedDSA Plan A decouples ONLY at the final projection layer. Mid-layer backbone features are already polluted by domain shift. 85+ downstream prototype/alignment experiments all fail because they operate on polluted features.
- **Must-solve**: (1) shallow decouple; (2) Plan A is ceiling; (3) need sharper+lighter than FDSE's DSE; (4) style/semantic separation theory not connected to FL pipeline.
- **Non-goals**: prototype routing, new trainables >10% backbone, backbone resize, VLM/CLIP, K > 4.
- **Constraints**: AlexNet/ResNet-18 backbone, FedBN BN locality, cos²+HSIC Plan A stays, 3-seed {2,15,333}, must beat PACS Plan A 82.31% by +0.5% (→ 82.81%), code <150 lines, 0 new trainable.
- **Success**: PACS AVG Best >= 82.81%; Office >= 83.05%; R=20 sanity shows per-layer cos²(IN_l, BN_l) drops from 0.5+ to <0.3; <150 LOC; 0 new trainable.

## Technical Gap

Plan A's `L_orth = cos²(z_sem, z_sty)` at final 128d projection. Backbone conv/BN layers receive only indirect gradient → weak decouple signal at front → mid-features polluted. All 85+ downstream methods operate on polluted features → stuck at Plan A ceiling.

FDSE uses per-layer DFE(shared)+DSE(local-erase) with ~DSE=DFE/94 new params. **HIBD differentiation**: 0 new params, only normalization scheme change.

IBN-Net (Pan 2018 ECCV): Instance Normalization channels naturally separate style; Batch Normalization channels preserve content. IN+BN parallel (IBN-b type) gives +2-3% DG in single-machine experiments. **We port IBN to FedBN + add explicit orthogonal loss**.

## Method Thesis

Upgrade FedDSA's "final-layer cos²" to "**hierarchical IBN-orthogonal decouple**" — replace 2 early BN layers with IBN (IN and BN parallel), explicitly enforce cos² orthogonality between IN-channel and BN-channel outputs at each IBN layer. Style (IN channel) separates from semantic (BN channel) early in backbone, not just at the end.

- NO optional supporting contribution (avoid sprawl).

## Proposed Method

### Complexity Budget
- Reused/frozen: backbone conv/linear, Plan A cos²+HSIC at final layer, sem_head/style_head, classifier, FedBN locality
- New: 2 BN layers → IBN (same overall params due to InstanceNorm affine=False, BN on half channels); 1 new loss term (L_orth_layer); 2 new hyperparameters (num_ibn, λ_orth_layer)
- 0 NEW TRAINABLE PARAMETERS

### IBN Module

```python
class IBN(nn.Module):
    def __init__(self, channels, in_ratio=0.5):
        super().__init__()
        self.in_ch = int(channels * in_ratio)
        self.bn_ch = channels - self.in_ch
        self.instance_norm = nn.InstanceNorm2d(self.in_ch, affine=False)  # 0 params
        self.batch_norm = nn.BatchNorm2d(self.bn_ch)                      # usual gamma/beta
    def forward(self, x):
        x_in, x_bn = torch.split(x, [self.in_ch, self.bn_ch], dim=1)
        out_in = self.instance_norm(x_in)
        out_bn = self.batch_norm(x_bn)
        return torch.cat([out_in, out_bn], dim=1)
```

### Core Mechanism — Per-layer Orthogonal Loss

For each IBN layer l, take spatial mean pool of IN/BN channel outputs, then cos² orthogonality:

```
in_feat_l = global_avg_pool(out_in_l)     # [B, C_in]
bn_feat_l = global_avg_pool(out_bn_l)     # [B, C_bn]
# both L2 normalized along channel dim
cos_sim_l = (normalize(in_feat_l) * normalize(bn_feat_l)).sum(dim=1)  # [B]
L_orth_layer_l = (cos_sim_l ** 2).mean()
L_orth_layer = (1/N_IBN) * sum_l L_orth_layer_l
```

Total loss:
```
L = L_CE + 1.0 * L_orth_end(Plan A) + 0.0 * L_HSIC + 0.5 * L_orth_layer(NEW)
```
