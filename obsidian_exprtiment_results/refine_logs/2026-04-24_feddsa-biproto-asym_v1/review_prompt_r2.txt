# FedDSA-BiProto: Federated Domain Prototype as a First-Class Shared Object for Cross-Domain Learning

## Problem Anchor (unchanged from round-0)

- Bottom-line: Under AlexNet from scratch FL cross-domain (PACS / Office-Caltech10), 3-seed {2,15,333}×R200 mean AVG Best must strictly beat FDSE local reproduction (PACS > 79.91, Office > 90.58). Current orth_only: PACS 80.64 ✅ (+0.73), Office 89.09 ❌ (−1.49).
- Must-solve: Office must recover −1.49pp plus ≥+0.5pp further; PACS must not regress (≥80.91).
- Non-goals: no dataset swap, no diagnostic paper, no module pile-up, no pretraining, no backbone swap.
- Constraints: 4090/3090 GPUs, ≤50 GPU-h pilot hard, ≤120 total incl ablation; AlexNet + FedBN; PACS E=5 / Office E=1; 1 week.
- Success: 3-seed mean AVG Best PACS ≥ 80.91 AND Office ≥ 91.08; no Last regression; 3 visual evidence suites delivered.

## Thesis (single sentence)

**Domain should be modeled as a first-class federated prototype object Pd, dual to class prototype Pc, with their mutual exclusion enforced in low-dimensional prototype space — this replaces feature-level adversarial/MI-minimization routes (CDANN/FedDP/FedSeProto), which are falsified in our setup.**

## Contribution Focus

- **Dominant (C1, ONLY headline)**: Federated Domain Prototype Pd ∈ ℝ^{D×d_z} with prototype-level class-domain cosine exclusion. First to model domain as a first-class federated shared object dual to class. Mechanism: on-the-fly batch-local class/domain centroids pushed orthogonal in proto space; EMA Pc/Pd as stable cross-round references.
- **Enabling infrastructure (NOT a co-equal contribution)**: Asymmetric statistic encoder (~1M MLP on (μ,σ) stats). Provides proper input representation to Pd; structurally cannot encode class discriminative signal.
- **Explicit non-contributions**: not claiming MI-optimal, not claiming communication efficiency, no convergence proof, no cross-backbone, no DP, **not claiming trunk decontamination**.

## Method

### Complexity Budget
- encoder_sem (60M, inherit orth_only, unchanged): trained by L_CE + L_CE_aug + L_orth + L_proto_excl (via batch class centroid)
- semantic_head, sem_classifier (inherit)
- BN running stats local (FedBN)
- style bank (μ,σ) AdaIN inherit
- **encoder_sty (new, ~1M)**: 2-layer MLP + LayerNorm on concat of (μ,σ) from conv1-3 pre-activation taps. Taps `.detach()` so L_sty_proto does NOT backprop to encoder_sem.
- **Pc (new, derived)**: EMA class centroid from z_sem, no-backward buffer
- **Pd (new, core)**: EMA domain centroid from z_sty, no-backward buffer
- **L_sty_proto (new)**: InfoNCE(z_sty, Pd, domain) + 0.5·MSE(z_sty, stopgrad(Pd[d])). Trains encoder_sty for domain discrimination.
- **L_proto_excl (new)**: cos²(batch_class_centroid(z_sem), batch_domain_centroid(z_sty)) over all (class, domain) pairs. Has real gradient to both encoders; EMA Pc/Pd only for monitoring.
- Deleted: L_sem_proto, α-sparsity default off.

### Interface Specification

Statistic encoder input: concat of per-channel (μ, σ) from conv1-3 BN-post pre-ReLU, detach before input. Output: z_sty ∈ ℝ^128, L2 normalized.

Pc update (server-side, after each round):
```
For each class c:
  If any participating client has class c:
    aggregated_c = weighted_mean over participating clients of client_class_c_mean
    Pc[c] ← m · Pc[c] + (1-m) · F.normalize(aggregated_c)
  Else:
    Pc[c] ← Pc[c]  (no-op)
Broadcast Pc.
```

Pd update: identical structure, indexed by domain (d) instead of class.

Gradient-flow table:
| Module | L_CE | L_CE_aug | L_orth | L_sty_proto | L_proto_excl |
|---|:-:|:-:|:-:|:-:|:-:|
| encoder_sem | ✅ | ✅ | ✅ | ❌ | ✅ (via batch class centroid) |
| semantic_head | ✅ | ✅ | ✅ | ❌ | ✅ |
| sem_classifier | ✅ | ✅ | ❌ | ❌ | ❌ |
| encoder_sty | ❌ | ❌ | ✅ | ✅ | ✅ (via batch domain centroid) |
| Pc, Pd | ❌ | ❌ | ❌ | ❌ (MSE uses stopgrad) | ❌ (EMA only) |
